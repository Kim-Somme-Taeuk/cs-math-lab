## 왜 배우나

<strong>그래프는 연결 관계를 표현하는 강력한 구조</strong>입니다. 도시 경로, 네트워크, 친구 관계,
패키지 의존성, 웹 링크를 모두 그래프로 볼 수 있습니다.

리스트나 배열은 주로 순서가 있는 데이터를 다루지만, 많은 문제는 "무엇과 무엇이 연결되어
있는가"가 핵심입니다. 그래프는 <strong>연결 자체를 데이터로 다루게 해 줍니다</strong>.

## 먼저 감 잡기

<strong>그래프는 정점과 간선으로 이루어집니다.</strong> <strong>정점은 대상</strong>이고, <strong>간선은 대상 사이의 연결</strong>입니다.

- 지하철 노선도: 역은 정점, 역 사이의 연결은 간선입니다.
- SNS: 사용자는 정점, 팔로우나 친구 관계는 간선입니다.
- 웹: 페이지는 정점, 링크는 간선입니다.
- 프로젝트 빌드: 파일이나 패키지는 정점, 의존 관계는 간선입니다.

간선에 방향이 있으면 <strong>방향 그래프</strong>, 방향이 없으면 <strong>무방향 그래프</strong>입니다. 간선에 비용이나
거리처럼 숫자가 붙으면 가중 그래프입니다.

## 핵심 정의

그래프 $G = (V, E)$는 <strong>정점 집합 V와 간선 집합 E</strong>로 구성됩니다.

무방향 그래프에서는 간선 `{A, B}`가 A와 B의 양방향 연결을 뜻합니다. 방향 그래프에서는
간선 `(A, B)`가 A에서 B로 가는 연결을 뜻하므로 `(B, A)`와 다릅니다.

그래프는 보통 <strong>두 방식으로 코드에 저장</strong>합니다.

| 표현 | 특징 |
| --- | --- |
| 인접 리스트 | 각 정점에서 갈 수 있는 이웃 목록을 저장 |
| 인접 행렬 | 정점 쌍마다 연결 여부를 표처럼 저장 |

## 자주 헷갈리는 지점

<strong>방향 그래프에서는 A에서 B로 가는 간선이 있어도 B에서 A로 가는 간선이 자동으로 생기지
않습니다.</strong> 팔로우 관계가 대표적인 예입니다.

<strong>경로와 간선도 구분해야 합니다.</strong> 간선은 직접 연결 하나이고, 경로는 여러 간선을 이어서 어떤
정점까지 가는 방법입니다. A -> B, B -> C가 있으면 A에서 C로 가는 경로는 있지만 A -> C
직접 간선이 있다고 말할 수는 없습니다.

DFS와 BFS도 자주 섞입니다.

- DFS는 한 길을 깊게 들어갔다가 돌아옵니다. 재귀나 스택과 잘 어울립니다.
- <strong>BFS는 가까운 정점부터 넓게 퍼집니다</strong>. 큐와 잘 어울리고, 가중치가 없는 그래프에서는 <strong>시작점으로부터의 최단 간선 수</strong>를 찾는 데 자주 씁니다.

<strong>방문 집합도 중요합니다.</strong> 이미 방문한 정점을 기록하지 않으면 사이클이 있는 그래프에서
탐색이 끝나지 않을 수 있습니다.

### 예제를 해석하는 순서

그래프 문제를 풀 때는 <strong>그림을 먼저 떠올리고, 그다음 코드 표현으로 옮깁니다</strong>.

1. 대상들을 정점으로 표시합니다.
2. 직접 연결만 간선으로 표시합니다.
3. 방향이 있는 연결인지 확인합니다.
4. "갈 수 있다"가 직접 간선인지, 여러 간선을 이은 경로인지 구분합니다.
5. 탐색 문제라면 시작점, 방문 순서, 방문 집합을 함께 적습니다.

예를 들어 A에서 B로, B에서 C로 갈 수 있으면 A에서 C로 가는 경로는 있습니다. 하지만
A에서 C로 가는 직접 간선이 있다고 말하려면 그 간선이 따로 주어져야 합니다.



## 코드로 읽기

```python
graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": [],
    "D": [],
}
```

이 객체는 <strong>인접 리스트</strong>입니다. `A`의 값이 `["B", "C"]`이므로 A에서 B와 C로 갈 수 있습니다.

BFS는 큐와 방문 집합을 함께 씁니다.

```python
from collections import deque

queue = deque(["A"])
visited = {"A"}

while queue:
    current = queue.popleft()

    for neighbor in graph[current]:
        if neighbor in visited:
            continue

        visited.add(neighbor)
        queue.append(neighbor)
```

여기서 `visited`는 집합 챕터와 연결됩니다. 이미 방문한 정점을 다시 큐에 넣지 않으므로
사이클이 있어도 탐색을 제어할 수 있습니다.

## 전공에서 만나는 장면

그래프는 다음처럼 작은 사례로 연결됩니다.

- DFS/BFS: 미로 탐색, 연결 요소 찾기, 웹 크롤링에 사용합니다.
- 최단 경로: 지도 앱은 장소를 정점, 도로를 간선으로 보고 비용이 작은 경로를 찾습니다.
- 의존성 그래프: 패키지나 작업 사이의 선후 관계를 그래프로 두고 설치 순서, 빌드 순서, 실행 순서를 정합니다.
- 추천 시스템: 사용자와 상품을 서로 다른 정점 집합으로 두고 클릭, 구매, 평점 간선을 분석합니다.
- 컴파일러: 모듈 import 관계를 그래프로 만들어 순환 의존성이나 먼저 처리해야 할 파일을 찾습니다.
