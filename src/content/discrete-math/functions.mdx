## 왜 배우나

집합 챕터에서는 "어떤 원소가 들어 있는가"를 보았습니다. 함수는 여기서 한 걸음 더 나아가
"입력 집합의 각 원소를 출력 집합의 어떤 원소에 연결할 것인가"를 묻습니다.

컴퓨터공학에서는 값을 그대로 보관하기보다 다른 형태로 바꾸는 일이 많습니다. 사용자 ID를
사용자 정보로 찾고, 문자열을 길이로 바꾸고, 점수를 등급으로 바꾸고, 파일 경로를 파일 내용과
연결합니다. 이런 장면을 읽는 기본 언어가 함수입니다.

다만 수학의 함수와 프로그래밍 언어의 함수는 완전히 같은 말이 아닙니다. 이 장에서는 먼저
수학의 함수 조건을 익히고, 그 뒤 코드에서 언제 함수처럼 읽어도 되는지 조심해서 연결합니다.

## 먼저 감 잡기

집합 A가 입력 후보이고 집합 B가 출력 후보라고 해 봅시다.

```txt
A = {alice, bob, chris}
B = {0, 1, 2}
```

함수는 A의 모든 원소를 B의 원소 정확히 하나에 대응시키는 규칙입니다. 예를 들어 다음 대응은
함수입니다.

```txt
alice -> 1
bob   -> 1
chris -> 2
```

`alice`와 `bob`이 같은 출력 `1`로 가도 괜찮습니다. 함수 조건은 "출력이 서로 달라야 한다"가
아니라 "각 입력의 출력이 정확히 하나여야 한다"입니다.

반대로 다음은 함수가 아닙니다.

```txt
alice -> 0
alice -> 2
bob   -> 1
chris -> 2
```

`alice`라는 같은 입력이 두 출력으로 갈라졌기 때문입니다.

## 핵심 정의

집합 A에서 집합 B로 가는 함수 $f: A \to B$는 A의 모든 원소 $a$에 대해 B의 원소 하나
$f(a)$를 대응시키는 규칙입니다.

| 이름 | 뜻 | 예시에서의 역할 |
| --- | --- | --- |
| 정의역 | 가능한 입력 전체 | `A = {alice, bob, chris}` |
| 공역 | 출력이 속하기로 한 전체 집합 | `B = {0, 1, 2}` |
| 치역 | 실제로 나온 출력들의 집합 | `{1, 2}` |

위 예시에서 공역은 `{0, 1, 2}`이지만 치역은 `{1, 2}`입니다. `0`은 출력 후보에는 들어 있지만
실제로는 아무 입력의 결과로도 나오지 않았기 때문입니다.

정의역, 공역, 치역을 코드식으로 말하면 이렇게 나눌 수 있습니다.

```txt
정의역: 이 규칙에 넣기로 약속한 입력 전체
공역: 결과가 들어가기로 약속한 출력 전체
치역: 실제로 계산해 보니 나온 결과들
```

예를 들어 $f(n) = n \bmod 2$이고 정의역이 정수 집합이라면, 공역을 `{0, 1}`로 둘 수 있고
치역도 `{0, 1}`입니다. 하지만 정의역을 `{2, 4, 6}`으로 제한하면 실제 출력은 항상 `0`이므로
치역은 `{0}`입니다.

## 자주 헷갈리는 지점

첫 번째 오해는 "여러 입력이 같은 출력으로 가면 함수가 아니다"입니다. 그렇지 않습니다.
학생 여러 명의 학년이 모두 1학년일 수 있고, 여러 문자열의 길이가 모두 5일 수 있습니다.
입력마다 출력이 하나라면 함수일 수 있습니다.

두 번째 오해는 "출력 후보를 모두 써야 함수다"입니다. 함수가 되기 위해 공역의 모든 원소가
반드시 나올 필요는 없습니다. 공역은 가능한 출력의 울타리이고, 치역은 실제로 나온 출력입니다.

세 번째 오해는 "프로그래밍 함수는 항상 수학 함수다"입니다. 수학 함수는 같은 입력이면 같은
출력이 나와야 합니다. 하지만 코드의 함수는 시간, 랜덤 값, 외부 상태, 데이터베이스 내용,
파일 시스템에 따라 같은 입력에도 다른 결과를 낼 수 있습니다.

```python
def pure_length(text):
    return len(text)
```

`pure_length("logic")`은 언제 호출해도 `5`입니다. 수학 함수처럼 읽기 쉽습니다.

```python
import random

def lucky_score(user_id):
    return random.randint(1, 100)
```

`lucky_score("u1")`은 같은 입력에도 호출할 때마다 달라질 수 있습니다. 프로그래밍 언어에서는
함수 형태로 작성했지만, 수학 함수처럼 "입력 하나가 출력 하나로 고정된다"고 보기 어렵습니다.

### 예제를 해석하는 순서

어떤 대응이 함수인지 판단할 때는 다음 순서가 안전합니다.

1. 입력 집합, 즉 정의역이 무엇인지 정합니다.
2. 각 입력에 출력이 하나씩 있는지 확인합니다.
3. 한 입력이 두 개 이상의 출력으로 갈라지는지 확인합니다.
4. 서로 다른 입력이 같은 출력으로 가는 경우를 문제로 착각하지 않습니다.
5. 실제 출력만 모아 치역을 따로 적어 봅니다.

이 순서는 코드 리뷰에서도 유용합니다. "이 함수가 모든 입력에서 값을 돌려주는가?", "같은
입력에서 결과가 안정적인가?", "외부 상태 때문에 결과가 달라지는가?"를 차례로 묻는 습관으로
이어집니다.

## 코드로 읽기

먼저 자연어 상황을 정합니다.

- 입력 집합: 문자열들
- 출력 집합: 0 이상의 정수들
- 규칙: 각 문자열을 그 길이에 대응시킨다

수학적으로는 문자열 `s`를 `len(s)`에 보내는 함수처럼 읽을 수 있습니다.

```python
def length_of(text):
    return len(text)

names = ["Ada", "Grace", "Linus"]
lengths = [length_of(name) for name in names]
```

여기서 `length_of`는 입력 문자열 하나를 길이 하나에 대응시킵니다. 같은 문자열을 넣으면 같은
길이가 나오므로 수학 함수와 연결하기 쉽습니다. 리스트 컴프리헨션은 이 규칙을 여러 입력에
차례로 적용해 출력 목록을 만듭니다.

이번에는 집합에서 이어지는 예시를 봅시다.

```python
words = {"logic", "sets", "proof"}
lengths = {len(word) for word in words}
```

`words`의 각 원소를 길이에 대응시키면 실제 출력들의 집합은 `{4, 5}`입니다. `logic`과 `proof`는
둘 다 길이 `5`로 갑니다. 여러 입력이 같은 출력으로 가지만, 각 입력의 출력은 하나이므로
함수 조건은 깨지지 않습니다.

ID 조회도 함수처럼 읽을 수 있습니다.

```python
users = {
    "u1": "Ada",
    "u2": "Grace",
}

def name_by_id(user_id):
    return users[user_id]
```

정의역을 `{"u1", "u2"}`로 제한하면 각 ID는 이름 하나에 대응됩니다. 하지만 정의역에 `"u3"`까지
포함한다고 말하려면 `"u3"`의 출력도 정해야 합니다. 그렇지 않으면 수학에서 말하는 전체 함수가
아니라 일부 입력에서만 정의된 규칙에 가깝습니다.

TypeScript/JavaScript에서도 같은 관점을 쓸 수 있지만, 언어 문법보다 대응 관계를 먼저 봅니다.

```ts
const lengthOf = (text: string) => text.length;
```

이 예시는 문자열을 숫자로 대응시키는 순수한 규칙에 가깝습니다. 반대로 다음처럼 외부 상태를
읽으면 같은 입력이라도 결과가 달라질 수 있습니다.

```ts
let bonus = 0;

const scoreOf = (base: number) => base + bonus;
```

`scoreOf(10)`은 `bonus` 값이 바뀌면 다른 결과를 냅니다. 코드에서는 함수지만, 수학 함수처럼
다루려면 어떤 상태까지 입력으로 볼지 따로 정해야 합니다.

## 전공에서 만나는 장면

함수는 전공 과목에서 다음처럼 나타납니다.

- 자료구조: 해시 함수는 키를 해시값으로 보내고, 구현에서는 그 해시값을 이용해 버킷 위치를 정할 수
  있습니다. 서로 다른 키가 같은 버킷으로 갈 수 있으므로 충돌 처리가 필요합니다.
- 데이터베이스: 기본키로 행을 찾는 규칙은 보통 키에서 행으로 가는 함수처럼 설계됩니다.
- 컴파일러: 소스 코드를 토큰 목록, AST, 중간 표현으로 단계적으로 바꾸는 변환을 함수 관점으로
  볼 수 있습니다.
- 타입 시스템: 값이나 표현식에 타입을 대응시키는 규칙을 다룹니다. 실제 언어에서는 문맥과
  추론 규칙이 함께 작동합니다.
- 상태 관리: 같은 현재 상태와 같은 액션에서 같은 다음 상태를 만드는 리듀서는 함수처럼
  테스트하기 쉽습니다.

단사, 전사, 전단사는 나중에 더 자세히 다룰 수 있습니다. 지금은 맛만 보면 됩니다.

- 단사: 서로 다른 입력이 같은 출력으로 겹치지 않습니다.
- 전사: 공역의 모든 원소가 실제 출력으로 한 번 이상 나옵니다.
- 전단사: 단사이면서 전사입니다. 입력과 출력이 빠짐없이 1:1로 맞물립니다.

이 세 개는 지금 당장 외울 대상이라기보다 "함수의 대응 모양을 더 자세히 분류하는 말" 정도로
두면 충분합니다.
