## 왜 배우나

함수는 입력을 받아 하나의 출력을 정하는 규칙입니다. 프로그래밍의 함수, 데이터 변환,
해시, 매핑 과정을 이해하는 데 필요한 기본 언어입니다.

컴퓨터공학에서 함수는 단순히 `function` 키워드로 만든 코드만 뜻하지 않습니다. 사용자 ID를
사용자 정보로 찾는 규칙, 파일 경로를 파일 내용으로 대응시키는 규칙, 문자열을 길이로
바꾸는 규칙도 함수처럼 생각할 수 있습니다.

## 먼저 감 잡기

함수는 각 입력에 대해 출력이 정확히 하나 정해지는 대응입니다. 같은 입력을 넣었는데
출력이 여러 개로 갈라지면 함수라고 보기 어렵습니다.

예를 들어 다음은 함수로 볼 수 있습니다.

- 학생 ID -> 학생 한 명의 이름
- 문자열 -> 문자열의 길이
- 정수 n -> n을 2로 나눈 나머지
- 상품 ID -> 상품 상세 정보

반대로 "학생 이름 -> 학생 ID"는 이름이 같은 학생이 있을 수 있으므로 조심해야 합니다. 한
이름이 여러 ID에 대응될 수 있다면 그 규칙은 함수가 아닐 수 있습니다.

## 핵심 정의

집합 A에서 집합 B로 가는 함수 $f: A \to B$는 A의 모든 원소에 대해 B의 원소 하나를
대응시키는 규칙입니다.

여기서 A를 정의역, B를 공역이라고 부릅니다. 실제로 함수가 만들어 내는 출력들의 모임은
치역입니다.

```txt
정의역 A: 가능한 입력 전체
공역 B: 출력이 속하기로 한 전체 범위
치역: 실제로 나온 출력들의 모음
```

예를 들어 $f(n) = n \bmod 2$이고 정의역이 정수 집합이라면 공역을 `{0, 1}`로 둘 수 있고,
치역도 `{0, 1}`입니다.

## 자주 헷갈리는 지점

서로 다른 입력이 같은 출력으로 갈 수는 있습니다. 하지만 하나의 입력이 동시에 여러 출력에
연결되면 함수 조건을 만족하지 않습니다.

```txt
함수 가능:
alice -> 3
bob   -> 3

함수 아님:
alice -> 3
alice -> 5
```

또 자주 헷갈리는 부분은 "출력이 없을 수 있는 입력"입니다. 수학에서 $f: A \to B$라고
말했다면 A의 모든 원소에 출력이 있어야 합니다. 코드에서는 `undefined`를 반환하는 함수가
있지만, 이때는 출력 범위에 `undefined`를 포함한 함수로 보거나, 일부 입력에서만 정의된
부분 함수처럼 따로 해석해야 합니다.

순수 함수와 프로그래밍 함수도 구분해야 합니다. 수학적 함수는 같은 입력이면 항상 같은
출력을 기대합니다. 하지만 코드의 함수는 시간, 랜덤 값, 데이터베이스 상태에 따라 같은
입력에도 다른 결과를 낼 수 있습니다.

## 직접 실험하기

함수 매핑 시각화는 다음 단계에서 추가합니다. 지금은 입력 하나가 출력 하나에만 연결되는지
손으로 확인해 봅니다.

### 예제를 해석하는 순서

어떤 대응이 함수인지 판단할 때는 다음 순서가 안전합니다.

1. 입력 집합이 무엇인지 정합니다.
2. 각 입력마다 출력이 있는지 확인합니다.
3. 한 입력이 두 개 이상의 출력으로 갈라지는지 확인합니다.
4. 서로 다른 입력이 같은 출력으로 가는 경우를 문제로 착각하지 않습니다.

예를 들어 "학생 ID -> 이름"은 보통 함수입니다. ID 하나는 학생 한 명을 가리키고, 그 학생의
이름은 하나로 정해집니다. 반면 "이름 -> 학생 ID"는 동명이인이 있으면 한 이름이 여러 ID로
갈 수 있으므로 함수 조건을 깨뜨릴 수 있습니다.



## 코드로 읽기

```ts
const lengthOf = (text: string) => text.length;
```

같은 문자열 입력에는 항상 같은 길이 출력이 정해집니다. 그래서 `lengthOf`는 문자열 집합에서
자연수 집합으로 가는 함수처럼 볼 수 있습니다.

배열의 `map`은 함수 개념을 코드에서 직접 쓰는 대표적인 예입니다.

```ts
const names = ["Ada", "Grace", "Linus"];
const lengths = names.map((name) => name.length);
```

여기서 `(name) => name.length`는 각 이름을 길이로 바꾸는 규칙입니다. `map`은 이 규칙을
입력 집합의 각 원소에 적용해 새 출력 목록을 만듭니다.

API 코드에서도 같은 관점을 쓸 수 있습니다.

```ts
const userById = new Map([
  ["u1", { name: "Ada" }],
  ["u2", { name: "Grace" }],
]);
```

`userById.get("u1")`이 항상 같은 사용자 객체를 가리키도록 관리한다면, 사용자 ID에서 사용자
정보로 가는 함수처럼 읽을 수 있습니다.

## 전공에서 만나는 장면

함수는 다음처럼 세부 장면에서 쓰입니다.

- 프로그래밍 언어: 함수 호출은 입력 인자를 출력 값으로 바꾸는 규칙입니다.
- 해시 테이블: 키를 해시값으로 보내고, 해시값을 버킷 위치와 연결합니다.
- 컴파일러: 소스 코드를 토큰 목록, AST, 바이트코드로 단계적으로 변환합니다.
- 데이터 파이프라인: 원본 행을 정제된 행으로 바꾸는 변환 함수들이 이어집니다.
- 상태 관리: 현재 상태와 액션을 다음 상태로 바꾸는 리듀서는 함수처럼 설계됩니다.
