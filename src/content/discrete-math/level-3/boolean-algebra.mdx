## 왜 배우나

부울 대수는 0과 1, 참과 거짓을 계산 규칙으로 다룹니다. 조건문, 필터 조건, 비트 연산, 디지털 회로는 모두
부울 대수의 언어로 읽을 수 있습니다.

Level 1/2에서 논리 연산과 드모르간 법칙을 봤다면, 여기서는 그 규칙을 코드 조건식과 비트 감각으로 연결합니다.
핵심은 <strong>복잡한 조건식을 같은 의미의 더 단순한 형태로 읽는 법</strong>입니다.

카르노 맵이나 회로 최적화 심화는 다루지 않습니다.

## 먼저 감 잡기

`is_admin and is_active`는 두 조건이 모두 참이어야 참입니다.

`is_admin or is_owner`는 둘 중 하나라도 참이면 참입니다.

`not is_deleted`는 삭제되지 않은 항목만 고르는 조건입니다.

## 핵심 정의

### 0과 1

부울 대수에서는 0을 거짓, 1을 참으로 봅니다. 프로그래밍의 boolean 값과 직접 연결됩니다.

### AND, OR, NOT

AND는 둘 다 1일 때만 1입니다.

OR는 하나라도 1이면 1입니다.

NOT은 0과 1을 뒤집습니다.

```python
allowed = is_admin or (is_owner and is_active)
hidden = not visible
```

### 항등법칙

항등법칙은 값을 그대로 유지하는 규칙입니다.

```text
A AND 1 = A
A OR 0 = A
```

### 지배법칙

지배법칙은 한 값이 전체 결과를 바로 정하는 규칙입니다.

```text
A AND 0 = 0
A OR 1 = 1
```

### 멱등법칙

같은 조건을 두 번 써도 의미는 바뀌지 않습니다.

```text
A AND A = A
A OR A = A
```

### 흡수법칙

흡수법칙은 중복된 조건을 줄일 때 자주 보입니다.

```text
A OR (A AND B) = A
A AND (A OR B) = A
```

### 드모르간 법칙

드모르간 법칙은 NOT이 괄호 안으로 들어갈 때 AND와 OR가 바뀐다는 규칙입니다.

```text
NOT (A AND B) = (NOT A) OR (NOT B)
NOT (A OR B) = (NOT A) AND (NOT B)
```

## 자주 헷갈리는 지점

<strong>AND와 OR의 기준을 섞는 실수.</strong> AND는 모두 참, OR는 하나라도 참입니다.

<strong>NOT을 괄호 안으로 넣을 때 연산자를 안 바꾸는 실수.</strong> 드모르간에서는 AND와 OR가 서로 바뀝니다.

<strong>중복 조건을 그대로 두는 실수.</strong> `A and A`는 A와 같습니다.

<strong>항등법칙과 지배법칙을 섞는 실수.</strong> `A and 1`은 A지만, `A and 0`은 0입니다.

<strong>비트 연산과 논리 연산을 무조건 같다고 보는 실수.</strong> 감각은 비슷하지만 언어별 연산자와 평가 방식은 다를 수 있습니다.

## 코드로 읽기

### 조건식 단순화

```python
if is_active and is_active:
    show_user()
```

멱등법칙으로 `is_active` 하나만 남겨도 의미가 같습니다.

```python
if is_active:
    show_user()
```

### 드모르간으로 조건 뒤집기

```python
if not (is_admin or is_owner):
    deny()
```

같은 뜻으로 다음처럼 읽을 수 있습니다.

```python
if (not is_admin) and (not is_owner):
    deny()
```

### 비트 연산 맛보기

```python
mask = READ | WRITE
can_read = permissions & READ
```

비트 연산은 각 자리의 0/1에 AND, OR 같은 규칙을 적용합니다. 여기서는 세부 최적화보다 0/1 규칙이 코드로
이어진다는 점만 잡으면 됩니다.

## 전공에서 만나는 장면

- if 조건문 읽기
- 필터 조건 단순화
- 권한 플래그와 비트마스크
- SQL WHERE 조건
- 디지털 회로의 AND/OR/NOT 게이트
- 테스트 조건의 반례 찾기
