## 왜 배우나

유클리드 알고리즘은 두 정수의 최대공약수 gcd를 빠르게 구하는 방법입니다. 코딩테스트에서는 약분, 최소공배수,
주기 계산, 정수 문제 전처리에서 자주 등장합니다.

정수론 기초에서 gcd의 의미를 봤고, 모듈러 연산에서 나머지를 봤습니다. 이 장에서는 그 둘을 연결합니다.
핵심은 <strong>나머지를 반복해서 gcd 문제를 더 작은 gcd 문제로 바꾸는 감각</strong>입니다.

확장 유클리드 알고리즘은 맛보기로만 언급하고, 디오판토스 방정식이나 모듈러 역원 본격 풀이는 다루지 않습니다.

## 먼저 감 잡기

84와 30의 gcd를 구한다고 해 봅시다. 약수를 전부 나열하지 않아도 됩니다.

```text
84 = 30 x 2 + 24
30 = 24 x 1 + 6
24 = 6 x 4 + 0
```

나머지가 0이 되는 순간의 나누는 수 6이 gcd입니다.

## 핵심 정의

### gcd 복습

gcd(a, b)는 a와 b를 동시에 나누는 가장 큰 양수입니다.

작은 수에서는 약수를 나열해도 되지만, 입력이 커지면 모든 약수를 확인하는 방식은 비효율적입니다.

### 핵심 등식

유클리드 알고리즘의 핵심은 다음입니다.

$$
gcd(a, b) = gcd(b, a \bmod b)
$$

a와 b의 공약수는 b와 a를 b로 나눈 나머지도 나눕니다. 그래서 gcd 문제가 더 작은 수의 gcd 문제로 바뀝니다.
이 장에서는 엄밀한 증명보다 이 변환을 코드로 읽는 감각에 집중합니다.

### 종료 조건

반복하다가 b가 0이 되면 현재 a가 gcd입니다.

```text
gcd(24, 6) = gcd(6, 0)
```

이때 gcd는 6입니다.

### lcm과의 연결

gcd를 구하면 lcm도 쉽게 구할 수 있습니다.

$$
lcm(a, b) = \frac{a \times b}{gcd(a, b)}
$$

곱셈이 큰 언어에서는 overflow를 피하려고 `(a // gcd(a, b)) * b`처럼 계산 순서를 조정하기도 합니다.

## 자주 헷갈리는 지점

<strong>나머지가 아니라 몫을 다음 입력으로 넣는 실수.</strong> 다음 단계는 gcd(b, a mod b)입니다.

<strong>종료 조건을 잘못 잡는 실수.</strong> b가 0이면 현재 a가 답입니다.

<strong>입력 순서에 너무 신경 쓰는 실수.</strong> gcd(a, b)는 gcd(b, a)와 같아서, 큰 수를 앞에 두고 시작하면 편합니다.

<strong>lcm 계산에서 gcd를 빼먹는 실수.</strong> lcm은 단순히 a x b가 아니라 a x b / gcd(a, b)입니다.

<strong>확장 유클리드까지 한 번에 외우려는 실수.</strong> 지금 단계에서는 기본 gcd 반복 구조가 먼저입니다.

## 코드로 읽기

### 반복 구현

```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

한 줄 `a, b = b, a % b`가 핵심입니다. 이전 b가 다음 a가 되고, 이전 나머지가 다음 b가 됩니다.

### 재귀 구현

```python
def gcd(a, b):
    if b == 0:
        return a

    return gcd(b, a % b)
```

재귀 버전은 수식 `gcd(a, b) = gcd(b, a mod b)`를 거의 그대로 옮긴 형태입니다.

### lcm 구현

```python
def lcm(a, b):
    return a // gcd(a, b) * b
```

먼저 나눈 뒤 곱하면 큰 수 언어에서 overflow 위험을 줄일 수 있습니다.

## 전공에서 만나는 장면

유클리드 알고리즘은 다음 상황에서 자주 만납니다.

- 분수 약분
- 최소공배수 계산
- 주기나 반복 패턴 맞추기
- 격자와 정수 좌표 문제
- RSA 같은 암호 기초 맛보기
- 확장 유클리드 알고리즘으로 이어지는 정수 알고리즘

시간복잡도는 입력 숫자가 빠르게 줄어든다는 점 때문에 매우 좋습니다. 지금 단계에서는 매번 나머지를 취하면서
문제 크기가 크게 줄어든다는 감각이면 충분합니다.
