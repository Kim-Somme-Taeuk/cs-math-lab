## 왜 배우나

DAG와 위상 정렬은 작업 사이의 의존성을 순서로 바꾸는 도구입니다. 빌드 순서, 선수 과목, 패키지 설치,
작업 스케줄링처럼 "먼저 해야 하는 것"이 있는 문제에서 자주 등장합니다.

이 장의 핵심은 <strong>방향 그래프에서 사이클이 없을 때, 의존성을 깨지 않는 순서를 만드는 법</strong>입니다.
SCC 같은 고급 그래프 알고리즘은 다루지 않습니다.

## 먼저 감 잡기

예를 들어 `코드 작성 -> 테스트 -> 빌드 -> 배포`라는 의존성이 있으면 배포를 먼저 할 수 없습니다.

가능한 순서는 의존성을 지키면 됩니다. 어떤 문제에서는 가능한 순서가 하나가 아니라 여러 개일 수 있습니다.

## 핵심 정의

### 방향 그래프

방향 그래프는 간선에 방향이 있는 그래프입니다. `A -> B`는 A에서 B로 향하는 관계입니다.

의존성 문제에서는 보통 "A를 먼저 해야 B를 할 수 있다"는 뜻으로 읽습니다.

### DAG

DAG는 directed acyclic graph, 즉 사이클이 없는 방향 그래프입니다.

사이클이 있으면 `A를 하려면 B가 먼저 필요하고, B를 하려면 A가 먼저 필요하다` 같은 모순이 생깁니다.

### 진입 차수

진입 차수는 어떤 정점으로 들어오는 간선의 개수입니다.

진입 차수가 0인 정점은 아직 남은 의존성이 없으므로 지금 처리할 수 있습니다.

### 위상 정렬

위상 정렬은 모든 간선 `A -> B`에 대해 A가 B보다 앞에 오도록 정점을 나열하는 것입니다.

가능한 순서가 여러 개일 수 있습니다. 중요한 것은 정답 하나를 외우는 것이 아니라, 모든 의존성을 지키는지 확인하는 것입니다.

### 큐 기반 맛보기

큐 기반 위상 정렬은 다음 흐름으로 읽으면 됩니다.

```text
1. 진입 차수 0인 정점을 큐에 넣는다.
2. 큐에서 하나를 꺼내 결과에 추가한다.
3. 그 정점에서 나가는 간선을 지우고, 새로 진입 차수 0이 된 정점을 큐에 넣는다.
4. 모든 정점을 처리할 때까지 반복한다.
```

## 자주 헷갈리는 지점

<strong>방향을 반대로 읽는 실수.</strong> `A -> B`를 A가 B보다 먼저라는 의존성으로 정했다면 끝까지 같은 기준으로 읽어야 합니다.

<strong>가능한 순서가 하나뿐이라고 생각하는 실수.</strong> 독립적인 작업이 있으면 여러 위상 정렬 결과가 가능합니다.

<strong>사이클이 있는데 정렬하려는 실수.</strong> 사이클이 있으면 모든 의존성을 만족하는 순서가 없습니다.

<strong>진입 차수 갱신을 빼먹는 실수.</strong> 정점을 처리하면 그 정점에서 나가는 간선도 사라진 것으로 보고 갱신해야 합니다.

<strong>큐에 들어간 순서와 유일한 정답을 혼동하는 실수.</strong> 큐 처리 순서에 따라 다른 정답이 나올 수 있습니다.

## 코드로 읽기

### 진입 차수 만들기

```python
indegree = [0] * n
graph = [[] for _ in range(n)]

for a, b in edges:
    graph[a].append(b)
    indegree[b] += 1
```

`a -> b`라면 b로 들어오는 간선이 하나 늘어납니다.

### 큐 기반 위상 정렬

```python
from collections import deque

queue = deque(i for i in range(n) if indegree[i] == 0)
order = []

while queue:
    current = queue.popleft()
    order.append(current)

    for next_node in graph[current]:
        indegree[next_node] -= 1
        if indegree[next_node] == 0:
            queue.append(next_node)
```

결과 길이가 n보다 작으면 사이클이 있어서 모든 정점을 처리하지 못한 것입니다.

- 데이터 파이프라인 순서
