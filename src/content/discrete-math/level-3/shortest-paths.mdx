## 왜 배우나

최단 경로는 그래프에서 어떤 지점까지 가는 최소 비용을 구하는 문제입니다. 네트워크 라우팅, 지도 경로,
게임 맵 이동, 추천 그래프 탐색처럼 전공 곳곳에서 만납니다.

이 장의 핵심은 <strong>그래프의 거리 배열을 갱신하면서 출발점에서 각 정점까지의 최소 비용을 읽는 법</strong>입니다.
벨만-포드, 플로이드-워셜, A*는 다루지 않습니다.

## 먼저 감 잡기

가중치가 없는 그래프에서는 간선 하나를 모두 같은 비용 1로 봅니다. 이때는 BFS가 가까운 정점부터 차례대로 방문하므로
최단 거리를 구할 수 있습니다.

가중치가 있는 그래프에서는 간선마다 비용이 다릅니다. 이때는 현재까지 가장 짧은 후보를 먼저 확정하는 다익스트라
알고리즘을 맛보기로 봅니다.

## 핵심 정의

### 거리의 의미

그래프에서 거리는 출발점에서 어떤 정점까지 가는 비용입니다.

가중치가 없으면 보통 지나간 간선 수가 거리입니다. 가중치가 있으면 간선 비용의 합이 거리입니다.

### 가중치 없는 그래프: BFS

BFS는 가까운 정점부터 방문합니다. 그래서 모든 간선 비용이 1이면 처음 방문한 거리가 최단 거리입니다.

```python
dist[start] = 0
queue = deque([start])
```

### 가중치 있는 그래프: 다익스트라 맛보기

다익스트라는 현재까지 알려진 거리 중 가장 작은 정점을 먼저 꺼냅니다. 그 정점에서 갈 수 있는 이웃의 거리를 더 짧게
만들 수 있으면 갱신합니다.

음수 간선이 있으면 이 전제가 깨질 수 있으므로 여기서는 음수 간선을 다루지 않습니다.

### 거리 배열

거리 배열 `dist[v]`는 출발점에서 v까지 현재까지 찾은 최단 거리 후보입니다.

처음에는 출발점만 0이고, 나머지는 매우 큰 값으로 둡니다.

### 우선순위 큐 감각

다익스트라에서는 매번 거리가 가장 작은 정점을 빨리 꺼내야 합니다. 그래서 우선순위 큐를 씁니다.

이 장에서는 구현 세부보다 "가장 가까운 후보를 먼저 확정한다"는 감각을 우선합니다.

### 경로 복원 맛보기

거리만 필요한 경우도 있지만, 실제 경로가 필요하면 이전 정점을 저장합니다.

```python
prev[next_node] = current
```

도착점에서 prev를 거꾸로 따라가면 경로를 복원할 수 있습니다.

## 자주 헷갈리는 지점

<strong>BFS를 가중치 그래프에 그대로 쓰는 실수.</strong> 간선 비용이 다르면 간선 수가 적은 경로가 항상 싼 것은 아닙니다.

<strong>음수 간선을 다익스트라에 넣는 실수.</strong> 다익스트라는 음수 간선이 없는 상황에서 써야 합니다.

<strong>거리 배열 초기화를 빼먹는 실수.</strong> 출발점은 0, 나머지는 큰 값으로 시작해야 합니다.

<strong>갱신 조건을 반대로 쓰는 실수.</strong> 새 비용이 기존 비용보다 작을 때만 갱신합니다.

<strong>경로 복원과 거리 계산을 섞는 실수.</strong> 거리 배열은 비용이고, prev 배열은 실제 경로를 되짚기 위한 정보입니다.

## 코드로 읽기

### BFS 거리

```python
from collections import deque

dist = [-1] * n
dist[start] = 0
queue = deque([start])

while queue:
    current = queue.popleft()
    for next_node in graph[current]:
        if dist[next_node] == -1:
            dist[next_node] = dist[current] + 1
            queue.append(next_node)
```

처음 방문한 순간의 거리가 최단 거리입니다.

### 다익스트라 맛보기

```python
import heapq

dist = [float("inf")] * n
dist[start] = 0
heap = [(0, start)]

while heap:
    cost, current = heapq.heappop(heap)
    if cost > dist[current]:
        continue

    for next_node, weight in graph[current]:
        next_cost = cost + weight
        if next_cost < dist[next_node]:
            dist[next_node] = next_cost
            heapq.heappush(heap, (next_cost, next_node))
```

핵심은 더 짧은 경로를 찾을 때만 거리 배열을 갱신하는 것입니다.

### 경로 복원

```python
prev[next_node] = current
```

갱신할 때 이전 정점을 저장하면, 도착점에서 거꾸로 따라가 실제 경로를 만들 수 있습니다.

## 전공에서 만나는 장면

최단 경로는 다음 상황과 연결됩니다.

- 지도와 내비게이션
- 네트워크 라우팅
- 게임 맵 이동
- 추천 그래프에서 가까운 관계 찾기
- 작업 비용이 있는 상태 그래프 탐색
