## 왜 배우나

재귀와 점화식 심화는 <strong>재귀 함수의 실행 비용을 점화식으로 표현하기 위해</strong> 배웁니다.
Level 2에서는 점화식이 "이전 값으로 현재 값을 정의하는 규칙"이라는 점을 봤습니다. 여기서는 그 감각을
알고리즘 코드로 가져옵니다.

재귀 코드를 분석할 때는 함수가 몇 번 호출되는지, 각 호출에서 얼마나 일하는지, 입력 크기가 어떻게 줄어드는지
확인해야 합니다. 이 과정은 분할 정복, DFS, 동적 계획법, 백트래킹 분석으로 바로 이어집니다.

이 장의 핵심은 점화식을 수학적으로 깊게 푸는 것이 아닙니다. <strong>재귀 코드를 보면 점화식으로 바꿀 수 있다</strong>는
감각을 만드는 것입니다.

## 먼저 감 잡기

Level 2 점화식에서 본 기본 요소를 다시 정리합니다.

- 기저 사례: 더 작게 나누지 않고 바로 끝나는 조건
- 재귀 정의: 이전 값이나 작은 문제로 현재 값을 정의하는 규칙
- $T(n) = T(n - 1) + 1$: n에서 n - 1로 내려가며 매번 상수만큼 일함
- $T(n) = T(n - 1) + n$: n에서 n - 1로 내려가지만 매번 n에 비례해 일함

점화식은 답 자체가 아니라 규칙입니다. 알고리즘 분석에서는 그 규칙을 펼쳐서 비용이 어떤 모양으로 쌓이는지
봅니다.

```python
def count_down(n):
    if n == 0:
        return

    count_down(n - 1)
```

이 코드는 입력 크기가 한 번에 1씩 줄고, 한 호출에서 특별히 큰 일을 하지 않습니다. 그래서 대략
$T(n) = T(n - 1) + O(1)$로 읽습니다.

## 핵심 정의

재귀 코드에서 점화식을 만들 때는 네 가지를 봅니다.

1. <strong>재귀 호출 횟수</strong>: 함수 안에서 자기 자신을 몇 번 부르는가?
2. <strong>한 호출에서 하는 일</strong>: 재귀 호출을 제외하고 반복문, 복사, 병합 같은 비용이 있는가?
3. <strong>종료 조건</strong>: 어느 입력 크기에서 더 내려가지 않는가?
4. <strong>입력 크기 변화</strong>: n - 1로 줄어드는가, n / 2로 줄어드는가, 여러 갈래로 나뉘는가?

이 네 가지를 모으면 $T(n)$을 세울 수 있습니다.

| 코드 구조 | 점화식 | 비용 감각 |
| --- | --- | --- |
| 하나씩 줄어드는 재귀 | $T(n) = T(n - 1) + O(1)$ | $O(n)$ |
| 하나씩 줄고 매번 n만큼 일함 | $T(n) = T(n - 1) + O(n)$ | $O(n^2)$ |
| 반으로 줄어드는 재귀 | $T(n) = T(n / 2) + O(1)$ | $O(\log n)$ |
| 두 갈래 재귀 | $T(n) = 2T(n - 1) + O(1)$ | 보통 지수적으로 커짐 |
| 분할 정복 | $T(n) = 2T(n / 2) + O(n)$ | $O(n \log n)$ |

여기서 중요한 것은 공식 암기가 아니라 구조 구분입니다. 재귀 호출이 하나인지 둘인지, 입력이 하나씩 줄어드는지
절반으로 줄어드는지가 비용을 크게 바꿉니다.

## 점화식 펼치기

점화식을 세운 뒤에는 몇 단계만 펼쳐 봐도 비용 모양이 보입니다.

### 하나씩 줄고 상수만큼 일함

$$
T(n) = T(n - 1) + 1
$$

$$
T(n) = T(n - 2) + 1 + 1
$$

이 과정은 n번 정도 이어지므로 $O(n)$입니다.

### 하나씩 줄고 매번 n만큼 일함

```python
def slow_sum(values, n):
    if n == 0:
        return 0

    subtotal = 0
    for i in range(n):
        subtotal += values[i]

    return subtotal + slow_sum(values, n - 1)
```

한 호출에서 반복문이 n번 돌고, 다음 호출은 n - 1 크기로 내려갑니다.

$$
T(n) = T(n - 1) + n
$$

펼치면 $n + (n - 1) + (n - 2) + \cdots + 1$이므로 $O(n^2)$입니다.

### 반으로 줄어드는 재귀

$$
T(n) = T(n / 2) + 1
$$

n이 n / 2, n / 4, n / 8처럼 줄어듭니다. 몇 번 줄이면 1이 되는지 보는 문제라서 $O(\log n)$입니다.

### 잘못된 식도 걸러야 함

$T(n) = 2T(n) + \cdots$처럼 입력 크기가 줄어들지 않는 식은 분석 식으로 문제가 있습니다. 재귀가 같은 크기의
문제를 다시 부른다는 뜻이기 때문입니다. 실제 코드라면 종료 조건에 도달하지 못하거나, 점화식을 잘못 세운
것일 가능성이 큽니다.

## 코드로 읽기

### `sum_to(n)`: $O(n)$

```python
def sum_to(n):
    if n == 1:
        return 1

    return sum_to(n - 1) + n
```

재귀 호출은 하나이고, 입력 크기는 n - 1로 줄어듭니다. 한 호출에서 더하는 일은 상수 시간입니다.

$$
T(n) = T(n - 1) + O(1)
$$

그래서 시간복잡도는 $O(n)$입니다. 반환값 자체는 $1 + 2 + \cdots + n$이지만, 계산 과정은 호출 n개가 한 줄로
이어집니다.

### 재귀 이진 탐색: $O(\log n)$

```python
def binary_search(values, target, left, right):
    if left > right:
        return False

    mid = (left + right) // 2

    if values[mid] == target:
        return True

    if values[mid] < target:
        return binary_search(values, target, mid + 1, right)

    return binary_search(values, target, left, mid - 1)
```

코드에는 재귀 호출문이 두 군데 보이지만, 실제 실행 경로에서는 둘 중 하나만 호출됩니다. 그래서
$2T(n/2)$가 아니라 $T(n/2)$입니다.

$$
T(n) = T(n / 2) + O(1)
$$

남은 범위가 절반씩 줄어들기 때문에 $O(\log n)$입니다.

### naive 피보나치: $O(2^n)$ 감각

```python
def fib(n):
    if n <= 1:
        return n

    return fib(n - 1) + fib(n - 2)
```

한 호출이 다시 두 호출을 만듭니다. 게다가 같은 작은 값을 계속 다시 계산합니다.

$$
T(n) = T(n - 1) + T(n - 2) + O(1)
$$

정확한 증명은 뒤로 미뤄도 됩니다. 지금은 호출 트리가 가지처럼 퍼져서 $O(2^n)$에 가까운 비용이 된다는 감각이
중요합니다.

DP 피보나치는 중복 계산을 저장해서 다릅니다. 각 n값을 한 번씩만 계산하면 $O(n)$으로 내려갈 수 있습니다.

### 병합 정렬 구조: $O(n \log n)$

```python
def merge_sort(values):
    if len(values) <= 1:
        return values

    mid = len(values) // 2
    left = merge_sort(values[:mid])
    right = merge_sort(values[mid:])

    return merge(left, right)
```

크기 n/2 문제 두 개를 풀고, 마지막에 전체 n개를 합칩니다.

$$
T(n) = 2T(n / 2) + O(n)
$$

여기서는 마스터 정리를 공식처럼 쓰지 않습니다. 깊이는 $\log n$ 정도이고, 각 레벨에서 합치는 비용이 전체
$O(n)$ 정도라서 $O(n \log n)$이라고 읽습니다.

## 재귀 트리 감각

재귀 트리는 호출이 어떻게 퍼지는지 보는 그림입니다.

- 호출이 한 줄로 이어지는 경우: `sum_to(n)`처럼 깊이만 생깁니다.
- 호출이 가지처럼 퍼지는 경우: naive 피보나치처럼 호출 수가 빠르게 늘어납니다.
- 분할 정복: 병합 정렬처럼 각 레벨의 문제 수는 늘지만 문제 크기는 줄어듭니다.

전체 비용은 보통 <strong>깊이</strong>와 <strong>각 레벨의 비용 합</strong>으로 봅니다.

병합 정렬은 깊이가 $\log n$이고 각 레벨의 합치는 비용이 $O(n)$입니다. 그래서 $O(n \log n)$이라는 감각이
생깁니다. 이 정도 구조 감각이면 지금 단계에서는 충분합니다.

## 자주 헷갈리는 지점

<strong>재귀가 있으면 무조건 느리다는 오해.</strong> 재귀 이진 탐색은 $O(\log n)$입니다. 느린지 빠른지는 재귀
자체가 아니라 호출 구조와 중복 계산 여부가 결정합니다.

<strong>재귀 호출 1개와 2개의 차이.</strong> 한 호출이 한 호출만 만들면 한 줄로 이어질 수 있습니다. 한 호출이
두 호출을 만들면 호출 트리가 퍼질 수 있습니다.

<strong>종료 조건을 빼먹는 실수.</strong> 기저 사례가 없으면 점화식도 어디서 멈추는지 알 수 없습니다.

<strong>한 호출의 비용을 빼먹는 실수.</strong> 병합 정렬에서 `merge` 비용을 빼면 $2T(n/2)$만 남는데, 실제로는
각 레벨에서 합치는 비용이 중요합니다.

<strong>피보나치를 $O(n)$으로 착각하는 실수.</strong> naive 재귀 피보나치는 같은 값을 계속 다시 계산합니다.
반복문이나 DP로 한 번씩만 계산하는 피보나치와 구분해야 합니다.

## 전공에서 만나는 장면

재귀 코드와 점화식 연결은 다음 주제에서 계속 등장합니다.

- 분할 정복: 이진 탐색, 병합 정렬, 퀵 정렬 구조 분석
- DFS: 정점과 간선을 얼마나 방문하는지 분석
- 동적 계획법: 재귀식을 상태 전이식으로 바꾸고 중복 계산을 제거
- 백트래킹: 선택지가 몇 갈래로 퍼지는지 보고 탐색 공간을 추정
- 그래프 알고리즘: 재귀나 큐, 스택 기반 탐색 비용 분석
