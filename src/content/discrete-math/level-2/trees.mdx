## 왜 배우나

<strong>트리</strong>는 그래프 중에서도 <strong>계층 구조</strong>를 표현하기 좋은 형태입니다. 파일 시스템, DOM, AST, 탐색 트리,
힙처럼 컴공 곳곳에서 등장합니다.

## 먼저 감 잡기

폴더 안에 파일과 하위 폴더가 있고, 그 안에 다시 파일이 들어 있습니다. 한 위치에서 시작해 아래로
뻗어 가지만, 다시 자기 조상으로 돌아가는 사이클은 없습니다. 이것이 트리의 감각입니다.

## 핵심 정의

<strong>트리는 연결되어 있고 사이클이 없는 그래프</strong>입니다. 정점이 n개인 트리는 <strong>간선이 n-1개</strong>입니다.

루트 트리에서는 한 정점을 루트로 정하고 방향감 있게 봅니다.

그래프 기초에서 배운 정점과 간선을 그대로 쓰지만, 트리는 그중에서도 <strong>계층 구조를 만들 수 있는
특별한 그래프</strong>입니다. 연결이 끊기면 하나의 트리가 아니고, 사이클이 생기면 계층 구조가 깨집니다.

예를 들어 A-B-C가 한 줄로 연결되어 있으면 그래프이면서 트리입니다. 여기에 C-A 간선을 추가하면
사이클이 생기므로 트리가 아닌 그래프가 됩니다. 반대로 A-B와 C-D처럼 두 덩어리로 끊겨 있으면
각 덩어리는 트리일 수 있지만 전체는 하나의 트리가 아니라 <strong>숲</strong>입니다.

- 루트: 시작 정점
- 부모 / 자식 / 형제
- 리프: 자식이 없는 정점
- 깊이: 루트에서 얼마나 떨어져 있는가
- 높이: 아래로 가장 멀리 내려가는 거리
- 이진 트리: 각 노드의 자식이 최대 2개인 트리

순회:

- 전위: 현재 노드 먼저
- 중위: 주로 이진 트리에서 왼쪽, 현재, 오른쪽
- 후위: 자식들을 먼저 보고 현재 노드

## 자주 헷갈리는 지점

<strong>모든 연결 그래프가 트리는 아닙니다.</strong> 사이클이 있으면 트리가 아닙니다. 반대로 사이클이 없어도
연결되어 있지 않으면 하나의 트리가 아니라 숲으로 봅니다.

<strong>트리의 높이와 깊이는 기준이 다릅니다.</strong> 깊이는 루트에서 내려온 거리이고, 높이는 그 노드에서
아래로 얼마나 더 내려갈 수 있는지 봅니다.

전위, 중위, 후위 순회는 모두 "현재 노드를 언제 처리하는가"의 차이입니다. 중위 순회는 특히
<strong>이진 트리</strong>에서 왼쪽 부분, 현재 노드, 오른쪽 부분을 읽을 때 자연스럽습니다.

## 직접 실험하기

정점이 5개인 트리라면 간선은 4개입니다. 간선이 너무 많으면 사이클이 생길 수 있고, 너무 적으면
연결이 끊길 수 있습니다.

예:

- A-B
- A-C
- B-D
- B-E

이 그래프는 연결되어 있고 사이클이 없으므로 트리입니다.

## 코드로 읽기

상황: 간단한 트리를 인접 리스트로 표현하고 DFS 순회를 합니다.

```python
tree = {
    "A": ["B", "C"],
    "B": ["D", "E"],
    "C": [],
    "D": [],
    "E": [],
}

def preorder(node):
    result = [node]
    for child in tree[node]:
        result += preorder(child)
    return result
```

`preorder("A")`는 현재 노드를 먼저 보고 자식으로 내려갑니다. 이것이 <strong>전위 순회</strong>입니다.

## 전공에서 만나는 장면

트리는 계층 구조를 저장하고 탐색하는 장면에서 자주 등장합니다.

- 파일 시스템: 폴더 안에 하위 폴더와 파일이 들어가는 구조를 루트와 자식 노드로 표현합니다.
- DOM 트리: HTML 문서는 부모 요소와 자식 요소로 이루어진 트리라서, 브라우저가 화면을 구성할 때 이 구조를 따라갑니다.
- AST: 컴파일러나 인터프리터는 소스 코드를 구문 트리로 바꿔 연산 순서와 문법 구조를 분석합니다.
- 이진 탐색 트리: 왼쪽은 작은 값, 오른쪽은 큰 값이라는 규칙으로 검색 범위를 줄입니다.
- 힙: 부모와 자식 사이의 우선순위 규칙을 유지해 최솟값이나 최댓값을 빠르게 꺼냅니다.
- 재귀 구조: 트리는 각 자식도 다시 작은 트리처럼 볼 수 있어 재귀 함수로 탐색하기 좋습니다.
