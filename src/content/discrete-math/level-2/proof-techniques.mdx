## 왜 배우나

Level 1에서는 명제, 조건문, 집합, 함수, 관계, 귀납법을 따로 배웠습니다. Level 2의 증명 기법은
이 도구들을 사용해 "왜 항상 맞는가?"를 설명하는 방법입니다.

컴공에서는 정답을 구하는 코드만큼이나, 그 코드가 모든 입력에서 맞는지 설명하는 일이 중요합니다.
정렬 알고리즘이 항상 순서를 보장하는지, 권한 조건이 빠진 경우가 없는지, 자료구조의 연산이
약속한 성질을 유지하는지 확인하려면 증명 방식이 필요합니다.

이 장의 목표는 어려운 증명을 많이 쓰는 것이 아닙니다. 문제를 보고 "직접 보일까?", "대우가
쉬울까?", "반례를 찾는 문제일까?"처럼 접근 방법을 고르는 감각을 만드는 것입니다.

## 먼저 감 잡기

증명은 상대를 설득하는 글이 아니라, <strong>빠진 경우가 없도록 논리를 정리하는 절차</strong>입니다.

예를 들어 다음 주장들을 봅시다.

- 모든 짝수 n에 대해 n + 2도 짝수이다.
- 어떤 사용자가 관리자라면 편집 권한을 가진다.
- 모든 오름차순 정렬 리스트의 마지막 원소가 최솟값이다.
- 모든 양의 정수 n에 대해 `sum_to(n)`은 1부터 n까지의 합을 반환한다.

이 주장들은 생김새가 조금씩 다릅니다.

- 정의를 바로 펼치면 쉬운 주장
- 조건문 $P \to Q$ 꼴인 주장
- "모든"이 붙어 있어서 반례 하나로 깨지는 주장
- n에서 n+1로 이어지는 구조가 보이는 주장

증명 기법은 이 생김새에 맞는 도구를 고르는 일에서 시작합니다.

## 핵심 정의

### 직접 증명

<strong>직접 증명</strong>은 가정에서 출발해 결론까지 바로 이어 가는 방식입니다. 조건문 $P \to Q$를 보일 때
P가 참이라고 두고, 정의나 이미 아는 성질을 사용해 Q를 얻습니다.

예를 들어 "n이 짝수이면 n + 2도 짝수이다"를 보려면 짝수의 정의를 사용합니다.

- n이 짝수라고 가정합니다.
- 그러면 어떤 정수 k에 대해 n = 2k입니다.
- n + 2 = 2k + 2 = 2(k + 1)이므로 n + 2도 짝수입니다.

정의만 펼쳤을 때 결론이 자연스럽게 나오면 직접 증명이 먼저입니다.

### 대우 증명

$P \to Q$와 $\lnot Q \to \lnot P$는 같은 의미입니다. 결론 Q를 직접 보이기 어렵지만,
Q가 아니라고 했을 때 P도 아님을 보이기 쉬우면 대우를 사용합니다.

예를 들어 "n²이 짝수이면 n도 짝수이다"는 직접 보이기보다 대우가 쉽습니다.

- 대우: n이 홀수이면 n²도 홀수이다.
- n = 2k + 1이면 n² = 4k² + 4k + 1 = 2(2k² + 2k) + 1입니다.
- 따라서 n²은 홀수입니다.

그러므로 원래 명제도 참입니다.

### 모순 증명

모순 증명은 <strong>보이고 싶은 결론이 거짓이라고 가정한 뒤</strong>, 이미 참인 사실과 충돌하는 결과를
끌어내는 방식입니다.

구조는 다음과 같습니다.

1. 결론이 거짓이라고 가정합니다.
2. 그 가정에서 논리를 전개합니다.
3. 불가능한 결과나 앞선 가정과 충돌하는 결과가 나옵니다.
4. 따라서 처음의 부정 가정이 틀렸고, 결론은 참입니다.

모순 증명은 강력하지만 초보자에게는 흐름이 헷갈릴 수 있습니다. 먼저 직접 증명이나 대우가
가능한지 보고, 막힐 때 선택하는 편이 안전합니다.

### 반례

"모든 A는 B이다"를 거짓으로 만들려면 A이지만 B가 아닌 예시 하나면 충분합니다. 이것을
<strong>반례</strong>라고 합니다.

예를 들어 "모든 소수는 홀수이다"는 거짓입니다. 2는 소수이지만 홀수가 아닙니다. 반례는
긴 설명보다 강합니다. 단, 반례는 주장이 "모든"을 말할 때만 잘 맞습니다.

### 귀납법 복습

자연수 n에 대한 명제가 있고, 작은 경우에서 큰 경우로 이어지는 구조가 보이면 귀납법을
고려합니다.

- 기저 사례: 가장 작은 n에서 참인지 확인합니다.
- 귀납 가정: P(k)가 참이라고 가정합니다.
- 귀납 단계: P(k+1)이 참임을 보입니다.

재귀 함수, 반복 횟수, 수열, 트리처럼 크기가 하나씩 커지는 대상은 귀납법과 잘 맞습니다.

## 자주 헷갈리는 지점

대우와 역은 다릅니다. $P \to Q$의 대우는 $\lnot Q \to \lnot P$이고, 역은 $Q \to P$입니다.
대우는 원래 명제와 같은 의미지만, 역은 반드시 같은 의미가 아닙니다.

모순 증명은 "아무 말이나 해서 이상한 결론을 만든다"가 아닙니다. 부정 가정에서 출발해
이미 참이라고 인정한 사실과 충돌해야 합니다.

반례는 주장을 증명하지 않습니다. 반례는 주장이 거짓임을 보일 때 쓰는 도구입니다.
반대로 예시 몇 개가 맞는다고 해서 "모든 경우"가 증명되지는 않습니다.

귀납법의 귀납 가정도 조심해야 합니다. P(k)를 가정하는 것은 P(k+1)을 그냥 믿는다는 뜻이
아닙니다. P(k)를 이용해 P(k+1)을 새로 보여야 합니다.

## 코드로 읽기

증명 기법은 코드의 검증 관점과 연결됩니다. 먼저 자연어로 주장을 분해합니다.

상황: 비어 있지 않은 리스트가 오름차순으로 정렬되어 있다면, 첫 원소는 모든 원소 이하입니다.

수학적 분해:

- P: 리스트가 비어 있지 않고 오름차순으로 정렬되어 있다.
- Q: 첫 원소가 모든 원소 이하이다.
- 목표: P가 참이면 Q가 참임을 보인다.

Python으로는 다음처럼 검사할 수 있습니다.

```python
def is_sorted(values):
    pairs = zip(values, values[1:])
    return all(left <= right for left, right in pairs)

def first_is_min(values):
    if not values:
        return False

    first = values[0]
    return all(first <= value for value in values)
```

이 코드는 증명 자체가 아닙니다. 하지만 증명에서 다루는 두 문장 P와 Q가 코드에서 어떤 조건으로
읽히는지 보여 줍니다. 직접 증명은 `bool(values)`와 `is_sorted(values)`가 모두 참일 때 왜
`first_is_min(values)`도 참인지 설명하는 일입니다.

반례를 찾는 사고도 코드로 표현할 수 있습니다. 이번에는 거짓 주장인 "모든 오름차순 정렬
리스트의 마지막 원소가 최솟값이다"를 봅니다.

```python
def last_is_min(values):
    if not values:
        return False

    last = values[-1]
    return all(last <= value for value in values)

def is_counterexample(values):
    return bool(values) and is_sorted(values) and not last_is_min(values)
```

`is_counterexample([1, 2, 3])`는 참입니다. 오름차순으로 정렬되어 있지만 마지막 원소 3은
최솟값이 아니기 때문입니다. 반대로 반례를 못 찾았다고 해서 자동으로 증명된 것은 아닙니다.

## 전공에서 만나는 장면

증명 기법은 다음 장면에서 자주 나옵니다.

- 알고리즘 정확성: 정렬, 탐색, 그래프 알고리즘이 항상 원하는 결과를 내는지 보입니다.
- 테스트 설계: 어떤 입력이 반례가 될 수 있는지 생각합니다.
- 보안 조건: 권한 검사에서 빠진 경우가 없는지 조건문을 검토합니다.
- 타입과 명세: 함수가 약속한 입력과 출력 관계를 만족하는지 설명합니다.
- 자료구조 불변식: 삽입과 삭제 뒤에도 구조의 성질이 유지되는지 보입니다.
