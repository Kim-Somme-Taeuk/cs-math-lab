## 왜 배우나

관계 챕터에서는 원소 사이의 연결을 배웠습니다. 동치관계는 그중에서도 <strong>"같다고 묶을 수 있는" 관계</strong>입니다. 완전히 같은 값이 아니어도 같은 그룹으로 취급할 수 있으면 동치관계가 자주 등장합니다.

## 먼저 감 잡기

파일을 확장자로 묶어 봅시다. `main.py`와 `test.py`는 같은 Python 파일 그룹입니다. `logo.png`는
다른 그룹입니다. "확장자가 같다"는 관계는 파일들을 겹치지 않는 그룹으로 나눕니다.

여기서 핵심은 단순히 "비슷하다"가 아니라, <strong>같은 기준으로 안정적으로 묶인다</strong>는 점입니다. 한 파일은
정확히 하나의 확장자 그룹에 들어가고, 그룹끼리는 서로 겹치지 않습니다.

## 핵심 정의

관계 R이 동치관계가 되려면 <strong>세 성질을 모두 만족</strong>해야 합니다.

- 반사성: 모든 x에 대해 xRx
- 대칭성: xRy이면 yRx
- 추이성: xRy이고 yRz이면 xRz

동치관계가 있으면 각 원소는 자기와 동치인 원소들의 묶음, 즉 동치류에 들어갑니다. 전체 원소들은
겹치지 않는 동치류들로 나뉘는데, 이것을 분할이라고 합니다.

반대로 전체 집합을 겹치지 않는 그룹으로 나누면, <strong>"같은 그룹에 속한다"는 관계</strong>는 동치관계가
됩니다. 그래서 동치관계는 관계를 확인하는 문제이면서 동시에 <strong>그룹을 만드는 기준</strong>이기도 합니다.

## 자주 헷갈리는 지점

동치관계는 *비슷해 보인다*가 아닙니다. <strong>반사성, 대칭성, 추이성</strong>을 모두 확인해야 합니다.
특히 "친구 관계"는 대칭적일 수는 있지만 추이적이지 않을 수 있습니다. A와 B가 친구이고 B와 C가
친구라고 해서 A와 C가 반드시 친구는 아닙니다.

이런 관계로 그룹을 만들면 기준이 흔들립니다. A-B, B-C는 같은 묶음으로 보이는데 A-C는 같은 묶음이
아니라면, `{A, B, C}`를 하나의 그룹으로 묶어도 되는지 애매해집니다. <strong>추이성이 깨지면 그룹 경계가
안정적으로 정해지지 않습니다.</strong>

## 직접 실험하기

관계 챕터에서는 순서쌍을 보고 성질을 판별했습니다. 동치관계에서는 한 단계 더 나아가, 관계가
원소들을 어떤 그룹으로 나누는지 봅니다. 동치관계가 있으면 전체 집합은 겹치지 않는 동치류들로
분할됩니다.

예: 집합 `{1, 2, 3, 4, 5, 6}`에서 "2로 나눈 나머지가 같다"는 관계

- 1, 3, 5는 같은 그룹입니다.
- 2, 4, 6은 같은 그룹입니다.
- 전체 집합은 `{1, 3, 5}`, `{2, 4, 6}`으로 나뉩니다.

아래 실험에서는 직접 그룹을 만들어 보며 <strong>분할</strong>과 <strong>동치류</strong>가 어떻게 연결되는지 확인합니다.

## 코드로 읽기

상황: 파일 확장자가 같으면 같은 그룹으로 묶습니다.

```python
def extension(filename):
    return filename.split(".")[-1]

def same_extension(left, right):
    return extension(left) == extension(right)
```

`same_extension(a, b)`는 반사성, 대칭성, 추이성을 만족합니다. 그래서 확장자별 분류는
<strong>동치류를 만드는 일</strong>로 볼 수 있습니다.

예를 들어 `same_extension("main.py", "test.py")`와 `same_extension("test.py", "tool.py")`가
참이면, `main.py`와 `tool.py`도 같은 확장자 그룹에 있어야 합니다. 이것이 코드 예시에서 보이는
추이성입니다.

## 전공에서 만나는 장면

동치관계는 "같은 것으로 취급해도 되는 기준"을 정할 때 자주 쓰입니다.

- 사용자 그룹화: 같은 조직, 같은 권한 등급, 같은 실험군처럼 사용자를 겹치지 않는 묶음으로 나눕니다.
- 캐시 키: 서로 다른 요청이라도 같은 결과를 내는 조건이면 같은 캐시 그룹으로 묶어 재사용할 수 있습니다.
- 파일 확장자별 분류: 파일 이름은 달라도 확장자가 같으면 같은 처리 규칙을 적용하는 그룹으로 봅니다.
- 유니온 파인드: 여러 원소가 같은 연결 그룹에 속하는지 빠르게 판정하는 자료구조입니다.
- 동일성 판정: 객체가 완전히 같은 주소인지, 같은 값인지, 같은 사용자로 볼 수 있는지처럼 "같음"의 기준을 정합니다.
