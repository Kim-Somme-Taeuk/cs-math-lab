## 왜 배우나

명제 논리는 P, Q처럼 이미 참/거짓이 정해지는 문장을 다룹니다. 하지만 전공에서는 "모든
사용자가 활성 상태인가?", "어떤 주문에 오류가 있는가?"처럼 여러 대상에 대해 조건을 검사하는
문장이 자주 나옵니다. <strong>술어 논리</strong>는 이런 문장을 논리식으로 읽는 도구입니다.

## 먼저 감 잡기

학생 목록이 있다고 합시다.

- "모든 학생이 과제를 제출했다"는 전체를 검사합니다.
- "어떤 학생이 과제를 제출했다"는 하나라도 있는지 검사합니다.

Python의 `all`, `any`와 매우 비슷한 감각입니다.

명제 논리의 P는 이미 하나의 참/거짓 문장이지만, 술어 논리의 `Active(x)`는 <strong>x에 무엇을 넣는지에
따라 결과가 달라지는 조건</strong>입니다. 그래서 술어 논리에서는 항상 "어떤 대상 범위에서 검사하는가",
즉 정의역을 같이 봐야 합니다.

## 핵심 정의

<strong>술어</strong>는 대상이 들어가야 참/거짓이 정해지는 문장입니다. 예를 들어 `Active(x)`는 x가 누구인지
정해져야 참/거짓이 됩니다.

- 정의역: x가 움직이는 대상의 범위
- 술어 $P(x)$: x에 따라 참/거짓이 달라지는 조건
- 전칭 수량자 $\forall x$: 모든 x에 대해
- 존재 수량자 $\exists x$: 어떤 x가 존재해서

예:

- $\forall x P(x)$: 모든 x가 P를 만족한다.
- $\exists x P(x)$: P를 만족하는 x가 적어도 하나 있다.

수량자의 부정:

- $\lnot \forall x P(x) \equiv \exists x \lnot P(x)$
- $\lnot \exists x P(x) \equiv \forall x \lnot P(x)$

코드로 읽으면 $\forall$는 `all(...)`, $\exists$는 `any(...)`와 가깝습니다. 다만 논리식에서는
항상 <strong>정의역이 무엇인지</strong>를 먼저 정해야 합니다. `users`를 검사하는지, 활성 사용자만 검사하는지,
관리자만 검사하는지에 따라 같은 술어도 다른 문장이 됩니다.

## 자주 헷갈리는 지점

<strong>"모든"의 부정은 "아무도 아니다"가 아니라 "적어도 하나는 아니다"</strong>입니다. 예를 들어
"모든 사용자가 활성 상태다"의 부정은 "활성 상태가 아닌 사용자가 적어도 한 명 있다"입니다.

<strong>중첩 수량자는 순서가 중요할 수 있습니다.</strong> "모든 사용자에게 어떤 파일이 있다"와 "어떤 파일이
모든 사용자에게 있다"는 같은 말이 아닙니다.

또한 "모든"이 들어간 문장이 참인지 보려면 <strong>반례 하나</strong>를 찾는 방식이 자주 편합니다. 모든 사용자가
활성 상태라는 주장은 비활성 사용자 한 명만 있어도 깨집니다.

## 직접 실험하기

다음 사용자 목록을 봅니다.

| user | active | admin |
| --- | --- | --- |
| Alice | true | false |
| Bob | true | true |
| Chris | false | false |

- 모든 사용자가 활성 상태인가? 아니오. Chris가 반례입니다.
- 어떤 사용자가 관리자 권한을 가지는가? 예. Bob이 있습니다.
- 관리자가 아닌 사용자가 존재하는가? 예. Alice와 Chris가 있습니다.

아래 실험에서 Chris의 활성 상태를 바꾸면 <strong>전칭 명제</strong>와 <strong>존재 명제</strong>의 결과가 어떻게 달라지는지
바로 확인할 수 있습니다.

## 코드로 읽기

상황: 사용자 목록에서 전체 조건과 존재 조건을 확인합니다.

```python
users = [
    {"name": "Alice", "active": True, "admin": False},
    {"name": "Bob", "active": True, "admin": True},
    {"name": "Chris", "active": False, "admin": False},
]

all_active = all(user["active"] for user in users)
some_admin = any(user["admin"] for user in users)
```

`all_active`는 <strong>$\forall x Active(x)$</strong>와 대응되고, `some_admin`은 <strong>$\exists x Admin(x)$</strong>와 대응됩니다.
참고로 Python의 `all(...)`은 빈 목록에서 `True`를 반환합니다. 이 장에서는 <strong>검사할 대상이 있는
경우</strong>를 기준으로 전칭과 존재를 연결해서 봅니다.
