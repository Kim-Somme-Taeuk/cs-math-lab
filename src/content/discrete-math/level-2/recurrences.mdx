## 왜 배우나

재귀 함수와 반복문은 현재 값을 이전 값으로부터 만들 때가 많습니다. <strong>점화식</strong>은 이런 구조를 수식으로 읽는 방법입니다. 알고리즘 시간복잡도와 동적 계획법을 이해하려면 점화식을 읽는 감각이 필요합니다.

## 먼저 감 잡기

`a_n = a_{n-1} + 3`은 이전 값에 3을 더해 다음 값을 만든다는 뜻입니다. 시작값이 `a_1 = 2`이면
수열은 2, 5, 8, 11처럼 이어집니다.

## 핵심 정의

<strong>점화식은 이전 항을 이용해 다음 항을 정의하는 식</strong>입니다.

- 기저 사례: 시작값
- 재귀 정의: 이전 값에서 다음 값을 만드는 규칙
- $a_n$: n번째 항
- $T(n)$: 입력 크기 n에서의 실행 시간이나 비용

대표 예:

- 피보나치: $F(n) = F(n-1) + F(n-2)$
- 선형 증가: $T(n) = T(n-1) + 1$
- 반복문 합: $T(n) = T(n-1) + n$
- 분할 정복 맛보기: $T(n) = 2T(n/2) + n$

등차수열은 이전 항에 같은 값을 더하는 형태이고, 등비수열은 이전 항에 같은 값을 곱하는 형태입니다.
예를 들어 $a_n = a_{n-1} + 3$은 등차적 흐름이고, $a_n = 2a_{n-1}$은 등비적 흐름입니다.

분할 정복 예시는 단순화를 위해 <strong>n이 2로 잘 나누어지는 경우</strong>를 기준으로 읽습니다.

## 자주 헷갈리는 지점

<strong>점화식은 닫힌 공식이 아닙니다.</strong> `T(n) = T(n-1) + n`은 답을 바로 주는 식이 아니라, 이전 값을
통해 현재 값을 계산하는 규칙입니다. 필요하면 펼쳐서 합으로 바꿉니다.

<strong>기저 사례가 없으면 어디서 시작해야 할지 알 수 없습니다.</strong> 재귀 함수에 종료 조건이 필요한 것과
같습니다.

알고리즘 분석에서 $T(n)$은 실제 초 단위 시간이 아니라 <strong>입력 크기 n에 대한 비용의 모양</strong>을 나타내는
기호로 자주 씁니다. 그래서 점화식을 읽을 때는 "값을 정확히 계산한다"보다 "어떤 구조로 비용이
쌓이는가"를 먼저 봅니다.

## 직접 실험하기

$T(1) = 1$, $T(n) = T(n-1) + n$을 펼쳐 봅니다.

- $T(2) = T(1) + 2 = 3$
- $T(3) = T(2) + 3 = 6$
- $T(4) = T(3) + 4 = 10$

결국 $1 + 2 + 3 + \cdots + n$과 연결됩니다.

아래 실험에서 시작값과 규칙을 바꿔 보면서, <strong>이전 값에서 다음 값이 만들어지는 흐름</strong>과
`T(n)=T(n-1)+n`을 펼친 합을 같이 봅니다.

## 코드로 읽기

상황: 1부터 n까지 더하는 재귀 함수를 점화식으로 읽습니다.

```python
def sum_to(n):
    if n == 1:
        return 1

    return sum_to(n - 1) + n
```

이 코드는 $S(1)=1$, $S(n)=S(n-1)+n$과 대응됩니다. 코드의 <strong>종료 조건은 기저 사례</strong>이고, <strong>재귀 호출은 점화식의 이전 항</strong>입니다.
