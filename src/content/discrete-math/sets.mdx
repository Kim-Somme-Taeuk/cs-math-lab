## 왜 배우나

컴퓨터공학에서는 값을 하나씩만 다루지 않습니다. 사용자 목록, 검색 결과, 태그 집합,
그래프의 정점 집합처럼 여러 대상을 묶어서 다루는 일이 많습니다.

집합은 "무엇이 포함되는가"를 기준으로 데이터를 생각하게 해 줍니다. 이 관점은 중복 제거,
권한 비교, 검색 필터, 방문 여부 기록처럼 실무 코드에서도 자주 등장합니다.

## 먼저 감 잡기

집합은 서로 구분되는 원소들의 모음입니다. 순서는 중요하지 않고, 같은 원소를 여러 번
써도 한 번만 포함된 것으로 봅니다.

```ts
new Set(["logic", "sets", "logic"]);
// 결과는 {"logic", "sets"}처럼 중복을 제거한 모음으로 볼 수 있습니다.
```

일상 예시로 보면 더 쉽습니다.

- 수강생 집합: 어떤 학생이 수업에 포함되는지 묻습니다.
- 장바구니 상품 집합: 어떤 상품 종류가 담겼는지 묻습니다.
- 검색 결과 집합: 어떤 문서가 조건을 만족하는지 묻습니다.
- 방문한 페이지 집합: 어떤 URL을 이미 처리했는지 묻습니다.

## 핵심 정의

집합 A와 B가 있을 때 자주 쓰는 연산은 다음과 같습니다.

| 연산 | 표기 | 의미 |
| --- | --- | --- |
| 합집합 | $A \cup B$ | A 또는 B에 속한 원소 |
| 교집합 | $A \cap B$ | A와 B에 모두 속한 원소 |
| 차집합 | $A - B$ | A에는 있고 B에는 없는 원소 |
| 여집합 | $A^c$ | 전체집합 U에는 있지만 A에는 없는 원소 |

부분집합도 중요합니다. $A \subseteq B$는 A의 모든 원소가 B에도 들어 있다는 뜻입니다.
예를 들어 "사용자가 가진 권한이 필요한 권한을 모두 포함하는가?"는 부분집합 검사로 볼 수
있습니다.

## 자주 헷갈리는 지점

집합에서는 순서와 중복을 무시합니다. `[1, 2]`와 `[2, 1]`은 배열로는 다르지만, 집합으로는
같은 원소를 가진 같은 집합입니다. `[1, 1, 2]`도 집합으로 보면 `{1, 2}`입니다.

여집합은 반드시 전체집합 U가 있어야 의미가 정해집니다. "A가 아닌 것"이 어디까지인지
정해야 하기 때문입니다. 학생 명단 안에서 A가 아닌 것인지, 전체 사람 중 A가 아닌 것인지에
따라 결과가 달라집니다.

합집합과 교집합도 코드에서 헷갈리기 쉽습니다.

- "둘 중 하나라도 만족"이면 합집합입니다.
- "둘 다 만족"이면 교집합입니다.
- "A 조건은 만족하지만 B 조건은 제외"이면 차집합입니다.

검색 필터에서 "프론트엔드 또는 백엔드 태그"는 합집합이고, "프론트엔드이면서 React 태그"는
교집합입니다.

### 예제를 해석하는 순서

집합 문제는 "원소 하나를 집어넣어 보면 어디에 남는가?"로 판단하면 쉽습니다.

1. 전체집합 U를 먼저 확인합니다.
2. 원소 x가 A에 들어 있는지 묻습니다.
3. 원소 x가 B에 들어 있는지 묻습니다.
4. 연산 기호가 "둘 중 하나", "둘 다", "A만" 중 무엇을 요구하는지 확인합니다.

예를 들어 검색 결과 A와 북마크 B가 있을 때 `A - B`는 "검색은 되었지만 아직 북마크하지
않은 항목"입니다. 수식만 보면 추상적이지만, 원소 하나를 기준으로 질문하면 코드의
필터 조건과 바로 연결됩니다.

## 코드로 읽기

```ts
const a = new Set([1, 2, 3]);
const b = new Set([3, 4, 5]);

const union = new Set([...a, ...b]);
const intersection = new Set([...a].filter((item) => b.has(item)));
const difference = new Set([...a].filter((item) => !b.has(item)));
```

교집합은 "A를 순회하면서 B에도 있는 원소만 남기는 필터"로 구현할 수 있습니다. 차집합은
반대로 "A를 순회하면서 B에는 없는 원소만 남기는 필터"입니다.

권한 검사도 집합으로 읽을 수 있습니다.

```ts
const required = new Set(["read", "write"]);
const granted = new Set(["read", "write", "delete"]);

const canProceed = [...required].every((permission) => granted.has(permission));
```

`required`가 `granted`의 부분집합이면 필요한 권한을 모두 가진 것입니다.

## 전공에서 만나는 장면

집합은 다음처럼 작은 사례로 연결됩니다.

- 데이터베이스: `UNION`, `INTERSECT`, `EXCEPT`는 합집합, 교집합, 차집합에 가깝습니다.
- 검색: 여러 필터 조건의 결과를 합치거나 겹치는 항목만 남깁니다.
- 그래프 탐색: 이미 방문한 정점을 `visited` 집합에 넣어 중복 방문을 막습니다.
- 권한 시스템: 필요한 권한 집합이 사용자의 권한 집합에 포함되는지 검사합니다.
- 캐시: 이미 계산한 키의 집합을 두고 같은 작업을 반복하지 않습니다.

큰 범주로 "자료구조에 쓰인다"라고 외우기보다, 포함 여부를 빠르게 묻는 장면을 떠올리면
집합이 왜 필요한지 더 잘 보입니다.
