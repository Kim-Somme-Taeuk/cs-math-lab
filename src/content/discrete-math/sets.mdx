## 1. 이 개념이 필요한 이유

컴퓨터공학에서는 값을 하나씩만 다루지 않습니다. 사용자 목록, 검색 결과, 태그 집합,
그래프의 정점 집합처럼 여러 대상을 묶어서 다루는 일이 많습니다.

집합은 "무엇이 포함되는가"를 기준으로 데이터를 생각하게 해 줍니다.

## 2. 직관적인 설명

집합은 서로 구분되는 원소들의 모음입니다. 순서는 중요하지 않고, 같은 원소를 여러 번
써도 한 번만 포함된 것으로 봅니다.

```ts
new Set(["logic", "sets", "logic"]);
// 결과는 {"logic", "sets"}처럼 중복을 제거한 모음으로 볼 수 있습니다.
```

## 3. 정확한 정의

집합 A와 B가 있을 때 자주 쓰는 연산은 다음과 같습니다.

| 연산 | 표기 | 의미 |
| --- | --- | --- |
| 합집합 | $A \cup B$ | A 또는 B에 속한 원소 |
| 교집합 | $A \cap B$ | A와 B에 모두 속한 원소 |
| 차집합 | $A - B$ | A에는 있고 B에는 없는 원소 |
| 여집합 | $A^c$ | 전체집합 U에는 있지만 A에는 없는 원소 |

## 4. 헷갈리는 부분

집합에서는 순서와 중복을 무시합니다. `[1, 2]`와 `[2, 1]`은 배열로는 다르지만, 집합으로는
같은 원소를 가진 같은 집합입니다.

여집합은 반드시 전체집합 U가 있어야 의미가 정해집니다. "A가 아닌 것"이 어디까지인지
정해야 하기 때문입니다.

## 6. 손풀이 문제

전체집합 $U = \{1,2,3,4,5,6\}$, $A = \{1,2,3\}$, $B = \{3,4,5\}$라고 하자.

1. $A \cup B$를 구하세요.
2. $A \cap B$를 구하세요.
3. $A - B$를 구하세요.
4. $A^c$를 구하세요.

## 7. 코드 연결

```ts
const a = new Set([1, 2, 3]);
const b = new Set([3, 4, 5]);

const intersection = [...a].filter((item) => b.has(item));
```

교집합은 "A를 순회하면서 B에도 있는 원소만 남기는 필터"로 구현할 수 있습니다.

## 8. 전공 연결

집합은 데이터베이스의 조건 검색, 검색 엔진의 필터, 그래프의 방문 집합, 권한 시스템의
역할 비교에 직접 연결됩니다. SQL의 `INTERSECT`, `UNION`도 집합 연산의 관점으로 볼 수
있습니다.
