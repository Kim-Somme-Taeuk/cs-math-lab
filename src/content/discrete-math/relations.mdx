## 1. 이 개념이 필요한 이유

관계는 데이터 사이의 연결을 표현합니다. 친구 관계, 팔로우 관계, 테이블 간 외래키,
그래프의 간선은 모두 관계로 볼 수 있습니다.

## 2. 직관적인 설명

관계는 두 원소가 어떤 기준으로 연결되어 있는지 나타냅니다. 함수는 관계 중에서도 입력마다
출력이 하나로 정해지는 특별한 경우입니다.

## 3. 정확한 정의

집합 A와 B 사이의 관계 R은 곱집합 $A \times B$의 부분집합입니다.

## 4. 헷갈리는 부분

관계는 함수보다 느슨합니다. 하나의 원소가 여러 원소와 연결되어도 관계로는 문제가 없습니다.

## 5. 인터랙티브 예시

반사성, 대칭성, 추이성 시각화는 다음 단계에서 추가합니다.

## 6. 손풀이 문제

1. "같은 학번 앞 두 자리를 가진다"는 관계가 대칭적인지 생각해 보세요.
2. "A가 B를 팔로우한다"는 관계가 항상 대칭적인지 설명하세요.

## 7. 코드 연결

```ts
type Edge = [string, string];
const follows: Edge[] = [["alice", "bob"], ["bob", "chris"]];
```

## 8. 전공 연결

관계는 그래프, 데이터베이스, 권한 모델, 추천 시스템의 기본 표현입니다.
