## 왜 배우나

관계는 데이터 사이의 연결을 표현합니다. 친구 관계, 팔로우 관계, 테이블 간 외래키,
그래프의 간선은 모두 관계로 볼 수 있습니다.

함수는 입력 하나에 출력 하나가 정해져야 하지만, 현실의 연결은 그렇게 깔끔하지 않은 경우가
많습니다. 한 사람이 여러 사람을 팔로우하고, 한 글에는 여러 태그가 붙고, 한 학생은 여러
수업을 들을 수 있습니다. 이런 연결을 다루는 언어가 관계입니다.

## 먼저 감 잡기

관계는 두 원소가 어떤 기준으로 연결되어 있는지 나타냅니다. 함수는 관계 중에서도 입력마다
출력이 하나로 정해지는 특별한 경우입니다.

예를 들어 다음은 모두 관계입니다.

- 사용자 A가 사용자 B를 팔로우한다.
- 학생 S가 과목 C를 수강한다.
- 문서 D에 태그 T가 붙어 있다.
- 숫자 a가 숫자 b보다 작거나 같다.
- 파일 A가 파일 B를 import한다.

관계는 연결의 목록으로 생각할 수 있습니다. `(alice, bob)`이 팔로우 관계에 들어 있다면
alice가 bob을 팔로우한다는 뜻입니다.

## 핵심 정의

집합 A와 B 사이의 관계 R은 곱집합 $A \times B$의 부분집합입니다. 즉 A의 원소와 B의 원소를
짝지은 순서쌍 중 일부를 고른 것입니다.

```txt
A = {alice, bob}
B = {math, cs}
R = {(alice, cs), (bob, math), (bob, cs)}
```

같은 집합 A 위의 관계 $R \subseteq A \times A$에서는 다음 성질을 자주 봅니다.

| 성질 | 의미 |
| --- | --- |
| 반사성 | 모든 a에 대해 a가 자기 자신과 관계 있다 |
| 대칭성 | a가 b와 관계 있으면 b도 a와 관계 있다 |
| 추이성 | a가 b와 관계 있고 b가 c와 관계 있으면 a도 c와 관계 있다 |

## 자주 헷갈리는 지점

관계는 함수보다 느슨합니다. 하나의 원소가 여러 원소와 연결되어도 관계로는 문제가 없습니다.
팔로우 관계에서 alice가 bob과 chris를 모두 팔로우해도 이상하지 않습니다.

대칭성도 자주 헷갈립니다. "친구"는 보통 서로 친구라고 생각하므로 대칭적입니다. 하지만
"팔로우"는 한쪽만 할 수 있으므로 대칭적이지 않습니다.

추이성은 "연결이 이어지면 건너뛴 연결도 성립하는가"를 묻습니다.

- "작거나 같다"는 추이적입니다. `a <= b`이고 `b <= c`이면 `a <= c`입니다.
- "친구이다"는 보통 추이적이지 않습니다. A가 B의 친구이고 B가 C의 친구라고 해서 A가 C의
  친구라고 보장할 수 없습니다.
- "같은 생년이다"는 추이적입니다. A와 B가 같은 생년이고 B와 C가 같은 생년이면 A와 C도
  같은 생년입니다.

## 직접 판별하기

다음 관계를 직접 판별해 봅시다.

```txt
A = {1, 2, 3}
R = {(1, 1), (2, 2), (3, 3), (1, 2), (2, 1), (2, 3)}
```

- 반사성: `(1, 1)`, `(2, 2)`, `(3, 3)`이 모두 있으므로 만족합니다.
- 대칭성: `(2, 3)`은 있지만 `(3, 2)`가 없으므로 만족하지 않습니다.
- 추이성: `(1, 2)`와 `(2, 3)`은 있지만 `(1, 3)`이 없으므로 만족하지 않습니다.

### 예제를 해석하는 순서

관계의 성질은 이름을 외우기보다 반례를 찾는 방식으로 판단합니다.

1. 반사성: 자기 자신과 연결되지 않은 원소가 하나라도 있는가?
2. 대칭성: `(a, b)`는 있는데 `(b, a)`가 없는 경우가 있는가?
3. 추이성: `(a, b)`와 `(b, c)`는 있는데 `(a, c)`가 없는 경우가 있는가?

성질이 성립한다고 말하려면 모든 경우가 통과해야 합니다. 반대로 성립하지 않음을 보일 때는
반례 하나면 충분합니다. 이 관점은 테스트 작성과도 닮아 있습니다.



## 코드로 읽기

```python
follows = {
    ("alice", "bob"),
    ("alice", "chris"),
    ("bob", "chris"),
}
```

이 집합은 순서쌍의 집합처럼 볼 수 있습니다. `("alice", "bob")`은 alice가 bob을 팔로우한다는
관계 하나입니다.

대칭성을 검사하려면 모든 `(a, b)`에 대해 `(b, a)`가 있는지 확인하면 됩니다.

```python
is_symmetric = all((b, a) in follows for (a, b) in follows)
```

될 수 있습니다.

## 전공에서 만나는 장면

관계는 다음처럼 구체적인 장면에서 쓰입니다.

- 그래프: 정점 사이의 간선은 정점 집합 위의 관계입니다.
- 데이터베이스: 학생-수업 수강 테이블은 학생과 수업 사이의 관계입니다.
- 추천 시스템: 사용자-상품 클릭 기록은 관계 데이터입니다.
- 타입 시스템: 어떤 타입이 다른 타입의 하위 타입인지도 관계로 볼 수 있습니다.
- 파일 의존성: A가 B를 import한다는 관계를 따라 빌드 순서를 정할 수 있습니다.
