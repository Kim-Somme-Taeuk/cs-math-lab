## 왜 배우나

수학적 귀납법은 무한히 많은 경우를 한 번에 다루는 증명 방법입니다. 재귀 함수와 반복문이
왜 모든 입력 크기에서 맞는지 설명할 때 자주 등장합니다.

"n이 1일 때 맞다", "n에서 맞으면 n+1에서도 맞다"를 보이면 모든 자연수 n에 대해 맞다고
말할 수 있습니다. 이 구조는 재귀 호출이 가장 작은 입력에서 끝나고, 큰 입력은 작은 입력의
정답을 이용해 해결되는 방식과 닮아 있습니다.

## 먼저 감 잡기

첫 번째 계단에 설 수 있고, 어느 계단에 설 수 있으면 다음 계단으로 갈 수 있다면 모든
계단에 도달할 수 있습니다.

도미노로 생각해도 됩니다. 첫 도미노가 넘어지고, 어떤 도미노가 넘어지면 바로 다음 도미노도
넘어진다는 규칙이 있으면 전체 도미노가 넘어집니다.

중요한 점은 두 조건이 모두 필요하다는 것입니다.

- 첫 도미노가 넘어지지 않으면 아무 일도 시작되지 않습니다.
- 다음 도미노로 이어지는 규칙이 없으면 중간에서 멈출 수 있습니다.

## 핵심 정의

명제 $P(n)$에 대해 $P(1)$이 참이고, 임의의 k에 대해 $P(k) \to P(k+1)$이 참이면 모든
자연수 n에 대해 $P(n)$이 참입니다.

보통 증명은 다음 순서로 씁니다.

1. 기저 사례: $P(1)$ 또는 가장 작은 입력에서 성립함을 보입니다.
2. 귀납 가정: 임의의 k에 대해 $P(k)$가 참이라고 가정합니다.
3. 귀납 단계: 그 가정으로부터 $P(k+1)$이 참임을 보입니다.
4. 결론: 모든 n에 대해 $P(n)$이 참입니다.

## 자주 헷갈리는 지점

기저 사례와 귀납 단계 중 하나라도 빠지면 전체 주장을 보장할 수 없습니다.

귀납 가정은 "증명할 것을 그냥 믿는다"가 아닙니다. 임의의 k에서 성립한다고 가정했을 때
바로 다음 경우 k+1을 밀어 올릴 수 있는지 보는 장치입니다. 한 계단에 서 있다고 가정하고
다음 계단으로 갈 수 있음을 보이는 것과 같습니다.

또 다른 실수는 k에서 k+1로 가는 연결을 보이지 않고, 예시 몇 개만 확인하는 것입니다.
`n = 1, 2, 3`에서 맞는다고 해서 모든 n에서 맞는 것은 아닙니다. 귀납법은 예시 확인이 아니라
무한히 이어지는 연결 규칙을 증명하는 방법입니다.

코드에서는 종료 조건과 재귀 단계가 둘 다 필요합니다.

```ts
function factorial(n: number): number {
  if (n === 1) return 1;
  return n * factorial(n - 1);
}
```

`if (n === 1)`이 없으면 기저 사례가 빠진 것이고, `factorial(n - 1)`이 더 작은 문제로
줄어들지 않으면 귀납 단계가 제대로 작동하지 않습니다.

## 직접 실험하기

도미노 또는 계단 시각화는 다음 단계에서 추가합니다. 지금은 기저 사례와 귀납 단계를 직접
분리해서 읽어 봅니다.

### 예제를 해석하는 순서

귀납법 문제를 보면 다음 틀에 맞춰 빈칸을 채웁니다.

1. 증명하려는 문장을 $P(n)$으로 둡니다.
2. 가장 작은 n에서 $P(n)$이 참인지 직접 확인합니다.
3. $P(k)$가 참이라고 가정했을 때 사용할 수 있는 식을 적습니다.
4. 목표가 $P(k+1)$임을 분명히 쓰고, 식을 그 모양으로 바꿉니다.

예를 들어 합 공식에서는 $P(k)$가 `1 + 2 + ... + k = k(k+1)/2`입니다. $P(k+1)$은 여기에
`k + 1`을 더한 식이어야 합니다. 목표 모양을 먼저 써 두면 중간 계산이 어디로 가야 하는지
덜 헷갈립니다.



## 코드로 읽기

```ts
function sum(n: number): number {
  if (n === 1) return 1;
  return n + sum(n - 1);
}
```

이 코드를 귀납법으로 읽으면 다음과 같습니다.

- 기저 사례: `sum(1)`은 1을 반환합니다.
- 귀납 가정: `sum(k)`가 `1 + 2 + ... + k`를 올바르게 반환한다고 가정합니다.
- 귀납 단계: `sum(k + 1)`은 `(k + 1) + sum(k)`이므로 `1 + 2 + ... + k + (k + 1)`을 반환합니다.

반복문도 루프 불변식으로 비슷하게 설명할 수 있습니다.

```ts
let total = 0;
for (let i = 1; i <= n; i += 1) {
  total += i;
}
```

각 반복이 끝날 때 `total`이 `1 + 2 + ... + i`라는 사실을 유지하면, 마지막 반복 뒤에는 원하는
합이 들어 있습니다.

## 전공에서 만나는 장면

귀납법은 다음처럼 구체적으로 연결됩니다.

- 재귀 함수: 종료 조건과 재귀 호출의 올바름을 설명합니다.
- 루프 불변식: 반복이 진행되어도 유지되는 조건을 증명합니다.
- 자료구조: 트리의 높이, 노드 수, 재귀 순회 알고리즘을 증명합니다.
- 알고리즘 분석: 분할 정복 점화식이 모든 입력 크기에 대해 성립함을 보입니다.
- 형식 언어: 문자열 길이에 대한 성질을 길이 n에서 n+1로 확장해 증명합니다.
