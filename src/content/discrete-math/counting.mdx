## 왜 배우나

경우의 수는 가능한 선택지를 세는 방법입니다. 브루트포스 탐색, 비밀번호 경우의 수,
확률 계산의 출발점이 됩니다.

프로그램을 작성할 때 가능한 입력 조합이 얼마나 많은지 모르면, 단순 반복으로 풀 수 있는지
판단하기 어렵습니다. 경우의 수는 "얼마나 많이 시도해야 하는가"를 먼저 계산하게 해 줍니다.

## 먼저 감 잡기

순서를 구분하는지, 중복을 허용하는지에 따라 사용하는 공식이 달라집니다.

- 순서가 중요하면 순열입니다. 비밀번호 `AB`와 `BA`는 다릅니다.
- 순서가 중요하지 않으면 조합입니다. 팀원 `{A, B}`와 `{B, A}`는 같습니다.
- 같은 것을 다시 고를 수 있으면 중복 허용입니다. 4자리 PIN은 같은 숫자를 여러 번 쓸 수
  있습니다.
- 같은 것을 다시 고를 수 없으면 중복 불허입니다. 카드 5장을 뽑을 때 같은 카드는 두 번
  뽑을 수 없습니다.

## 핵심 정의

서로 다른 n개 중 r개를 순서 있게 고르면 순열, 순서 없이 고르면 조합입니다.

$$
P(n,r) = \frac{n!}{(n-r)!}
$$

$$
C(n,r) = \binom{n}{r}
$$

$$
\binom{n}{r} = \frac{n!}{r!(n-r)!}
$$

여기서는 서로 다른 n개 중에서 r개를 고르고, 0 ≤ r ≤ n인 경우를 다룹니다.

곱의 법칙도 자주 씁니다. 첫 번째 선택이 a가지, 두 번째 선택이 b가지이고 두 선택을 모두
해야 한다면 전체는 $a \times b$가지입니다.

합의 법칙은 둘 중 하나를 선택하는 경우에 씁니다. 서로 겹치지 않는 선택지가 a가지와 b가지
있다면 전체는 $a + b$가지입니다.

## 자주 헷갈리는 지점

"A, B"와 "B, A"를 같은 선택으로 볼지 다른 선택으로 볼지가 핵심입니다.

문제를 읽을 때 다음 질문을 순서대로 던지면 실수를 줄일 수 있습니다.

1. 순서가 결과를 바꾸는가?
2. 이미 고른 것을 다시 고를 수 있는가?
3. 모든 선택을 동시에 해야 하는가, 아니면 여러 갈래 중 하나를 고르는가?

예를 들어 "5명 중 회장과 부회장을 뽑는다"는 순서가 중요합니다. 회장 A, 부회장 B와 회장 B,
부회장 A는 다릅니다. 반면 "5명 중 2명을 팀원으로 뽑는다"는 순서가 중요하지 않습니다.

코드에서도 비슷합니다. 이중 반복문이 `(a, b)`와 `(b, a)`를 둘 다 만들면 순서 있는 선택을
세고 있는 것입니다.

## 코드로 읽기

```python
items = ["A", "B", "C"]

for a in items:
    for b in items:
        if a != b:
            print(a, b)
```

이 코드는 `(A, B)`와 `(B, A)`를 둘 다 방문하므로 순서 있는 선택을 셉니다. 서로 다른 n개에서
2개를 고르면 n × (n - 1)가지입니다.

순서 없는 쌍은 보통 인덱스를 이용해 뒤쪽 원소만 고릅니다.

```python
for i in range(len(items)):
    for j in range(i + 1, len(items)):
        print(items[i], items[j])
```

이렇게 하면 `(A, B)`를 세고 나서 `(B, A)`를 다시 세지 않습니다. 그래서 결과는 `nC2`가 됩니다.

완전 탐색의 시간 복잡도도 경우의 수와 연결됩니다. 10개의 원소에서 3개를 고르는 모든 조합을
검사하면 최대 `10C3 = 120`개를 확인합니다.

## 전공에서 만나는 장면

경우의 수는 다음처럼 세부 주제와 연결됩니다.

- 완전 탐색: 가능한 후보 개수를 세어 시간 안에 돌릴 수 있는지 판단합니다.
- 테스트: 여러 입력 조건의 조합 수를 계산해 테스트 범위를 조절합니다.
- 보안: 비밀번호 길이와 문자 종류로 가능한 비밀번호 수를 추정합니다.
- 확률: 가능한 전체 경우와 원하는 경우를 세어 확률을 계산합니다.
- 최적화: 가능한 선택 조합이 너무 많으면 동적 계획법이나 가지치기가 필요합니다.
