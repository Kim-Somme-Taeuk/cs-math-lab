## 1. 이 개념이 필요한 이유

경우의 수는 가능한 선택지를 세는 방법입니다. 브루트포스 탐색, 비밀번호 경우의 수,
확률 계산의 출발점이 됩니다.

## 2. 직관적인 설명

순서를 구분하는지, 중복을 허용하는지에 따라 사용하는 공식이 달라집니다.

## 3. 정확한 정의

서로 다른 n개 중 r개를 순서 있게 고르면 순열, 순서 없이 고르면 조합입니다.

$$
_nP_r = \frac{n!}{(n-r)!}, \quad _nC_r = \frac{n!}{r!(n-r)!}
$$

## 4. 헷갈리는 부분

"A, B"와 "B, A"를 같은 선택으로 볼지 다른 선택으로 볼지가 핵심입니다.

## 5. 인터랙티브 예시

순열과 조합 시뮬레이터는 다음 단계에서 추가합니다.

## 6. 손풀이 문제

1. 서로 다른 5개 중 2개를 순서 있게 고르는 경우의 수를 구하세요.
2. 서로 다른 5개 중 2개를 순서 없이 고르는 경우의 수를 구하세요.

## 7. 코드 연결

```ts
for (const a of items) {
  for (const b of items) {
    if (a !== b) {
      // 순서 있는 선택
    }
  }
}
```

## 8. 전공 연결

경우의 수는 완전 탐색의 시간 복잡도, 확률, 조합 최적화 문제로 이어집니다.
