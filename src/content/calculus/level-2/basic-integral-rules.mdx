## 왜 배우나

기본 적분 규칙은 간단한 누적량을 빠르게 읽기 위한 최소 도구입니다. 계산 훈련 자체가 목표가 아니라, 상수 함수나 거듭제곱 함수처럼 자주 나오는 형태의 누적량이 어떻게 생기는지 보는 것이 중요합니다.

이 장의 핵심은 <strong>기본 함수의 적분 규칙을 누적량 모델로 읽는 법</strong>입니다. 삼각함수, 로그, 지수 적분 공식 암기는 다루지 않습니다.

## 먼저 감 잡기

항상 5만큼의 비용이 3시간 동안 발생하면 총 비용은 15입니다. 상수 함수의 적분은 높이가 일정한 직사각형 넓이와 같습니다.

`x^2`처럼 높이가 점점 커지는 함수는 뒤쪽 구간의 기여량이 더 큽니다. 그래서 거듭제곱 함수의 적분은 단순히 같은 높이를 곱하는 것보다 다른 형태가 됩니다.

## 핵심 정의

기본 규칙은 다음 감각으로 충분합니다.

```text
상수 c의 적분: 일정한 높이가 쌓인다
x^n의 적분: x^(n+1)/(n+1) 꼴로 누적된다
합의 적분: 각각의 누적량을 더한다
상수배: 전체 누적량도 그만큼 커진다
```

부정적분의 `+ C`는 미분하면 사라지는 시작 높이 차이를 나타냅니다.

## 자주 헷갈리는 지점

<strong>적분상수 `C`를 장식으로 보는 실수.</strong> 같은 변화율을 가진 함수들이 서로 위아래로 이동할 수 있음을 나타냅니다.

<strong>공식을 외우고 누적 의미를 잊는 실수.</strong> 규칙은 누적량을 빠르게 구하기 위한 요약입니다.

<strong>정적분과 부정적분을 섞는 실수.</strong> 정적분은 구간의 누적값, 부정적분은 원시함수 가족을 찾는 관점입니다.

## 코드로 읽기

```python
def total_linear_cost(a, b, steps):
    width = (b - a) / steps
    total = 0
    for i in range(steps):
        x = a + (i + 0.5) * width
        total += (2 * x + 3) * width
    return total
```

공식을 쓰면 빠르게 계산할 수 있지만, 코드는 여전히 작은 기여량을 더하는 누적 구조를 보여 줍니다.

## 전공에서 만나는 장면

- 일정한 요청률을 시간에 따라 합산
- 단순 비용 함수를 구간 전체 비용으로 변환
- 물리 시뮬레이션에서 간단한 속도 함수 누적
- 공식 계산과 수치 근사의 결과 비교
