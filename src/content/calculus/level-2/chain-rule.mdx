## 왜 배우나

실제 프로그램과 모델은 함수 하나로 끝나지 않고 여러 변환이 이어진 구조입니다. 입력이 전처리를 거쳐 모델로 들어가고, 모델 출력이 손실 함수로 들어가는 식입니다. 이런 합성 구조의 변화율을 읽는 규칙이 연쇄법칙입니다.

이 장의 핵심은 <strong>함수 안에 함수가 들어간 계산 흐름에서 변화율이 어떻게 전달되는지 읽는 법</strong>입니다. 역전파 전체 유도와 자동미분 내부 구현은 다루지 않습니다.

## 먼저 감 잡기

`y = f(g(x))`에서는 x가 먼저 `g`를 지나고, 그 결과가 다시 `f`의 입력이 됩니다. x를 조금 바꾸면 `g`의 출력이 바뀌고, 그 변화가 다시 `f`의 출력 변화로 이어집니다.

연쇄법칙은 각 단계의 변화율을 이어 곱해 전체 변화율을 읽는 규칙입니다.

## 핵심 정의

합성함수는 한 함수의 출력이 다른 함수의 입력으로 들어가는 구조입니다.

```text
x -> g(x) -> f(g(x)) -> output
전체 변화율 = 바깥 변화율 * 안쪽 변화율
```

계산 그래프에서는 각 노드의 변화율이 다음 노드로 전달된다고 볼 수 있습니다.

## 자주 헷갈리는 지점

<strong>바깥 함수만 미분하는 실수.</strong> 안쪽 함수가 얼마나 변하는지도 함께 곱해야 합니다.

<strong>합성 순서를 거꾸로 읽는 실수.</strong> 데이터가 어떤 함수 순서로 흐르는지 먼저 봐야 합니다.

<strong>역전파 전체를 여기서 다 이해해야 한다고 생각하는 실수.</strong> 지금은 변화율이 단계별로 전달된다는 감각만 잡습니다.

```text
raw input -> normalize -> model -> loss
```

## 코드로 읽기

```python
z = normalize(x)
y = model(z)
loss = squared_error(y, target)
```

x가 조금 바뀌면 `normalize`의 출력 z가 바뀌고, z 변화가 모델 출력 y를 바꾸며, y 변화가 손실을 바꿉니다. 연쇄법칙은 이 연결된 변화율을 따라갑니다.
