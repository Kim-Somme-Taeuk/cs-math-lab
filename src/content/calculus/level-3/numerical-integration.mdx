## 왜 배우나

원시함수를 모르거나 데이터가 샘플로만 주어진 경우에도 누적량을 구해야 할 때가 많습니다. 수치적분은 직사각형이나 사다리꼴 같은 근사로 넓이와 총량을 계산합니다.

이 장의 핵심은 <strong>연속적인 누적량을 코드에서 작은 조각의 합으로 근사하는 법</strong>입니다. Simpson 공식과 오차항 이론은 다루지 않습니다.

## 먼저 감 잡기

직사각형 방법은 각 구간을 하나의 높이로 대표합니다. 사다리꼴 방법은 구간 양끝 높이를 연결해 평균 높이로 근사합니다.

step size가 작아지면 보통 더 정확해지지만 계산량이 늘어납니다.

## 핵심 정의

직사각형 방법은 `높이 × 폭`을 더합니다. 사다리꼴 방법은 양끝 높이의 평균에 폭을 곱합니다.

```text
rectangle = f(x) * dx
trapezoid = (f(left) + f(right)) / 2 * dx
```

## 자주 헷갈리는 지점

<strong>step size를 줄이면 공짜로 정확해진다고 보는 실수.</strong> 정확도는 좋아질 수 있지만 계산량도 늘어납니다.

<strong>샘플 데이터와 함수식을 같은 방식으로만 보는 실수.</strong> 실제 데이터는 불규칙한 간격이나 노이즈가 있을 수 있습니다.

<strong>근사를 정확값으로 착각하는 실수.</strong> 수치적분은 오차를 가진 계산입니다.

## 직접 판별하기

다음 중 수치적분이 어울리는 상황을 골라 봅니다.

- 센서가 0.1초마다 기록한 속도에서 이동 거리 추정
- 시간별 서버 비용률에서 총 비용 추정
- 샘플링된 전력 사용량에서 총 에너지 추정

## 코드로 읽기

```python
area = 0
for i in range(len(xs) - 1):
    width = xs[i + 1] - xs[i]
    height = (ys[i] + ys[i + 1]) / 2
    area += width * height
```

이 코드는 인접한 두 샘플을 사다리꼴로 보고 넓이를 누적합니다.
