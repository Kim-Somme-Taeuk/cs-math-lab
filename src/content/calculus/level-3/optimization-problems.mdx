## 왜 배우나

컴공에서 많은 문제는 “어떤 값을 가장 좋게 만들 입력을 찾는 일”로 바뀝니다. 손실을 줄이고, 비용을 낮추고, 게임 밸런스 점수를 목표에 가깝게 만드는 것이 모두 최적화 문제입니다.

이 장의 핵심은 <strong>목적 함수와 후보점을 정하고, 변화율로 더 좋은 방향을 찾는 법</strong>입니다. 라그랑주 승수와 볼록 최적화 심화는 다루지 않습니다.

## 먼저 감 잡기

최적화 문제에는 바꾸는 입력과 평가하는 함수가 있습니다. 입력을 바꿨을 때 목적 함수 값이 작아지면 최소화 문제에서는 더 좋아진 것입니다.

기울기가 0인 지점은 주변에서 증가/감소 방향이 바뀔 수 있는 후보점입니다. 하지만 항상 정답은 아니므로 맥락과 주변 값을 같이 봐야 합니다.

## 핵심 정의

최소화 문제는 목적 함수 값을 작게 만드는 입력을 찾는 문제입니다.

```text
목적 함수: loss(x)
목표: loss(x)가 작아지는 x 찾기
```

후보점은 최적값일 가능성이 있는 입력입니다. 미분 가능한 1차원 함수에서는 기울기 0인 지점이 중요한 후보가 됩니다.

## 자주 헷갈리는 지점

<strong>기울기 0이면 무조건 최솟값이라고 보는 실수.</strong> 최댓값이나 평평한 변곡점일 수도 있습니다.

<strong>무엇을 바꾸는지 정하지 않는 실수.</strong> 입력 변수와 목적 함수를 먼저 정해야 합니다.

<strong>최소화와 최대화를 섞는 실수.</strong> 같은 함수도 목표에 따라 해석이 달라집니다.

## 직접 판별하기

아래 상황에서 목적 함수와 바꿀 입력을 찾아봅니다.

- 모델 파라미터를 바꿔 예측 오차를 줄인다.
- 서버 수를 조절해 비용과 지연 시간을 낮춘다.
- 게임 아이템 수치를 조절해 승률 차이를 줄인다.

## 코드로 읽기

```python
def objective(x):
    return (x - 3) ** 2 + 2

candidates = [-1, 0, 2, 3, 4]
best = min(candidates, key=objective)
```

이 코드는 후보점 중 목적 함수가 가장 작은 입력을 고릅니다. 실제 최적화는 후보를 더 똑똑하게 만들기 위해 변화율을 사용합니다.

- 검색/추천 점수 최대화
- 게임 밸런스 파라미터 조정
