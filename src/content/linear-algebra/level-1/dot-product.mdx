## 왜 배우나

내적은 두 벡터를 곱해 하나의 숫자로 만드는 연산입니다. 단순 계산을 넘어서 두 벡터가 얼마나 비슷한 방향을 보는지,
서로 직각에 가까운지, 추천이나 검색에서 얼마나 비슷한지 판단하는 데 연결됩니다.

이 장의 핵심은 <strong>내적을 방향의 비슷함을 읽는 신호로 이해하는 것</strong>입니다. 정사영 공식 심화는 다루지 않습니다.

## 먼저 감 잡기

`[1, 2]`와 `[3, 4]`의 내적은 같은 위치끼리 곱해서 더합니다.

```text
1 x 3 + 2 x 4 = 11
```

내적이 크면 두 벡터가 비슷한 방향을 보는 경우가 많고, 0이면 직교라고 부릅니다. 검색과 추천에서는 벡터가 비슷한 방향을
보는지 비교하는 감각이 중요합니다.

## 핵심 정의

### 내적 계산

두 벡터의 같은 위치 성분끼리 곱한 뒤 모두 더합니다.

```text
[a, b] · [c, d] = ac + bd
```

결과는 벡터가 아니라 숫자 하나입니다.

### 방향이 비슷한 정도

내적은 두 벡터가 비슷한 방향을 볼수록 커지는 경향이 있습니다. 반대 방향이면 음수가 될 수 있습니다.

### 직교

내적이 0이면 두 벡터가 직교한다고 말합니다. 2D 그림에서는 서로 직각인 방향으로 볼 수 있습니다.

### 코사인 유사도 맛보기

코사인 유사도는 길이 차이를 줄이고 방향의 비슷함을 비교하려는 방법입니다. 검색, 추천, 임베딩 비교에서 자주 나옵니다.

## 자주 헷갈리는 지점

<strong>내적 결과를 벡터로 착각하는 실수.</strong> 내적 결과는 숫자 하나입니다.

<strong>성분끼리 곱한 뒤 더하지 않는 실수.</strong> `[1, 2] · [3, 4]`는 `[3, 8]`이 아니라 11입니다.

<strong>내적이 크면 무조건 좋은 추천이라고 보는 실수.</strong> 길이 영향이 있을 수 있어 코사인 유사도 같은 보정이 필요할 수 있습니다.

<strong>직교를 관련이 전혀 없다는 뜻으로만 보는 실수.</strong> 수학적으로는 내적이 0인 방향 관계입니다.

## 코드로 읽기

```python
def dot(u, v):
    return sum(u[i] * v[i] for i in range(len(u)))
```

임베딩 검색에서는 문서 벡터와 질문 벡터를 비교합니다.

```python
score = dot(query_embedding, document_embedding)
```

실제 시스템에서는 길이를 정규화한 코사인 유사도를 쓰는 경우가 많습니다.
