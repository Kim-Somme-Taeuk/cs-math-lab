## 왜 배우나

행렬은 숫자를 행과 열로 정리한 표입니다. 단순한 표를 넘어서 여러 벡터를 한 번에 묶거나, 데이터를 변환하는 규칙을
표현할 때 쓰입니다.

이 장의 핵심은 <strong>행렬을 숫자 표, 데이터 묶음, 변환 규칙이라는 세 관점으로 읽는 법</strong>입니다.
행렬식은 아직 다루지 않습니다.

## 먼저 감 잡기

이미지는 픽셀 값이 격자로 놓인 데이터입니다. 표 데이터는 행마다 하나의 샘플, 열마다 하나의 특징을 놓을 수 있습니다.

```text
[
  [1, 2, 3],
  [4, 5, 6]
]
```

위 행렬은 2행 3열입니다.

## 핵심 정의

### 행과 열

행은 가로줄이고, 열은 세로줄입니다.

```text
2 x 3 행렬 = 2행 3열 행렬
```

### 숫자 표

행렬은 데이터를 표처럼 정리합니다. 데이터 분석에서는 행을 샘플, 열을 특징으로 보는 경우가 많습니다.

### 벡터 여러 개 묶기

벡터를 여러 개 쌓으면 행렬이 됩니다.

```text
[1, 2]
[3, 4]
[5, 6]
```

세 개의 2차원 벡터를 묶은 표처럼 볼 수 있습니다.

### 변환 규칙

행렬은 벡터를 다른 벡터로 바꾸는 규칙도 표현합니다. 이 관점은 뒤의 행렬 곱셈과 선형변환에서 중요해집니다.

## 자주 헷갈리는 지점

<strong>행과 열을 뒤집는 실수.</strong> 2 x 3은 2행 3열입니다.

<strong>행렬을 무조건 어려운 공식으로만 보는 실수.</strong> 처음에는 숫자 표로 보면 충분합니다.

<strong>모든 행렬이 정사각형이라고 생각하는 실수.</strong> 2 x 3, 10 x 4처럼 직사각형 행렬도 많습니다.

<strong>행렬과 벡터의 관계를 끊어서 보는 실수.</strong> 행렬은 벡터 여러 개를 묶거나 벡터를 변환하는 데 쓰입니다.

cols = len(matrix[0])

dataset = [

    [height, weight, age],
]
