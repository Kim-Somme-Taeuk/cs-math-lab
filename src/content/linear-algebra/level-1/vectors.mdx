## 왜 배우나

벡터는 여러 숫자를 하나의 값처럼 묶어 다루는 도구입니다. 2D 좌표에서는 방향과 크기가 있는 화살표로 볼 수 있고,
데이터에서는 한 대상의 특징을 모은 숫자 묶음으로 볼 수 있습니다.

이 장의 핵심은 <strong>벡터를 계산 공식보다 위치, 방향, 특징을 표현하는 기본 단위로 읽는 법</strong>입니다.
벡터공간 공리 같은 엄밀한 정의는 다루지 않습니다.

## 먼저 감 잡기

게임 캐릭터가 오른쪽으로 3, 위로 2 움직인다면 `[3, 2]`라고 쓸 수 있습니다. 사용자의 키, 몸무게, 나이도
`[키, 몸무게, 나이]`처럼 묶을 수 있습니다.

좌표 평면에서는 `[3, 2]`를 원점에서 (3, 2)까지 향하는 화살표로 볼 수 있습니다. 머신러닝에서는 문장이나 이미지도
숫자 벡터로 바꿔 비교합니다.

## 핵심 정의

### 크기와 방향

2D나 3D에서는 벡터를 크기와 방향이 있는 값으로 볼 수 있습니다.

```text
[3, 2] = 오른쪽 3, 위쪽 2
```

### 숫자 묶음

데이터에서는 벡터를 특징들의 묶음으로 봅니다.

```python
user = [age, login_count, purchase_count]
```

각 칸은 어떤 특징 하나를 나타냅니다.

### 벡터와 점

점은 위치를 말하고, 벡터는 이동량이나 방향을 말합니다. 하지만 원점을 기준으로 보면 점 `(3, 2)`와 벡터 `[3, 2]`를
비슷하게 그릴 수 있어서 자주 섞입니다.

### 임베딩 맛보기

임베딩은 단어, 문장, 이미지 같은 대상을 숫자 벡터로 바꾼 것입니다. 숫자 벡터가 되면 비슷한 대상끼리 비교하거나
검색할 수 있습니다.

## 자주 헷갈리는 지점

<strong>벡터를 무조건 화살표로만 보는 실수.</strong> 화살표는 좋은 그림이지만, 데이터에서는 특징 벡터처럼 숫자 묶음으로 더 자주 봅니다.

<strong>점과 벡터를 완전히 같은 것으로 보는 실수.</strong> 점은 위치, 벡터는 이동량이라는 관점이 다릅니다.

<strong>숫자 개수의 의미를 놓치는 실수.</strong> `[3, 2]`는 2차원, `[3, 2, 5]`는 3차원 벡터입니다.

<strong>벡터의 각 칸이 무엇을 뜻하는지 정하지 않는 실수.</strong> 특징 벡터에서는 각 위치의 의미가 중요합니다.

## 코드로 읽기

```python
position = [3, 2]
velocity = [1, -1]
features = [0.8, 0.1, 0.4]
```

코드에서는 벡터를 리스트나 배열로 자주 표현합니다. 중요한 것은 숫자들이 어떤 순서와 의미로 들어 있는지입니다.

```python
image_pixel = [red, green, blue]
```

RGB 색상도 세 숫자로 된 벡터처럼 볼 수 있습니다.

## 전공에서 만나는 장면

벡터는 다음 상황과 연결됩니다.

- 2D/3D 좌표와 이동량
- 이미지의 RGB 값
- 유저나 상품의 특징 벡터
- 문장 임베딩과 검색
- 머신러닝 모델의 입력 데이터
