## 왜 배우나

행렬 곱셈은 행렬을 단순히 숫자 표로만 보지 않고, 벡터를 바꾸는 규칙과 그 규칙의 합성으로 읽게 해 줍니다.
그래픽스, 머신러닝, 데이터 변환에서 매우 자주 등장합니다.

이 장의 핵심은 <strong>행렬 곱셈을 차원이 맞을 때 가능한 변환 적용과 변환 합성으로 이해하는 것</strong>입니다.
고급 블록 행렬은 다루지 않습니다.

## 먼저 감 잡기

행렬-벡터 곱은 벡터 하나를 새 벡터로 바꿉니다.

```text
[[2, 1],
 [0, 1]] [3, 2] = [8, 2]
```

행렬-행렬 곱은 변환을 이어서 적용하는 것과 연결됩니다.

## 핵심 정의

### 행렬-벡터 곱

행렬의 각 행과 벡터를 내적해서 결과 벡터의 각 성분을 만듭니다.

```text
2 x 2 행렬 * 2차원 벡터 = 2차원 벡터
```

### 행렬-행렬 곱

행렬-행렬 곱은 왼쪽 행렬의 행과 오른쪽 행렬의 열을 내적해 새 행렬을 만듭니다.

```text
(m x n) * (n x k) = (m x k)
```

가운데 차원 n이 맞아야 합니다.

### 변환의 합성

행렬 A와 B를 곱한다는 것은 변환 B를 적용한 뒤 A를 적용하는 것처럼 읽을 수 있습니다. 순서가 중요합니다.

## 자주 헷갈리는 지점

<strong>행렬 곱셈을 성분별 곱으로 착각하는 실수.</strong> 행렬 곱셈은 행과 열의 내적입니다.

<strong>차원 확인을 나중에 하는 실수.</strong> 먼저 `(m x n) * (n x k)` 형태인지 봐야 합니다.

<strong>곱셈 순서가 바뀌어도 같다고 보는 실수.</strong> 행렬 곱셈은 보통 순서가 바뀌면 결과가 달라집니다.

<strong>행렬-벡터 곱과 행렬-행렬 곱을 끊어서 보는 실수.</strong> 둘 다 변환 적용이라는 관점으로 이어집니다.

## 코드로 읽기

```python
def mat_vec_mul(A, x):
    return [
        sum(A[row][col] * x[col] for col in range(len(x)))
        for row in range(len(A))
    ]
```

라이브러리에서는 보통 다음처럼 씁니다.

```python
y = A @ x
z = A @ B
```

`@`는 Python/NumPy에서 행렬 곱셈을 나타냅니다.
