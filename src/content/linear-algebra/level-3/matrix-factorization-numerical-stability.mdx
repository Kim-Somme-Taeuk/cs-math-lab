## 왜 배우나

실제 수치계산에서는 큰 선형시스템을 역행렬로 직접 풀지 않는 경우가 많습니다. 계산량, 부동소수점 오차, 조건수 때문에 더 안정적인 분해 방법이 필요합니다.

핵심은 <strong>수학적으로 같은 식처럼 보여도 컴퓨터에서는 어떤 방식으로 계산하느냐가 정확도와 안정성을 바꾼다는 것</strong>입니다.

## 먼저 감 잡기

`x = A^-1 b`라고 쓰면 간단해 보이지만, 실제 구현에서는 `solve(A, b)`나 LU/QR 분해를 씁니다. 역행렬을 만들면 오차와 비용이 커질 수 있습니다.

## 핵심 정의

### 행렬 분해

복잡한 행렬을 다루기 쉬운 행렬들의 곱으로 나누는 방법입니다.

### LU 분해

선형시스템을 단계적으로 풀기 좋게 행렬을 아래삼각/위삼각 구조로 나누는 감각입니다.

### QR 분해

직교 방향과 삼각 구조로 나누는 방식입니다. 최소제곱이나 안정적인 계산에서 자주 등장합니다.

### 조건수

입력의 작은 변화가 출력에 얼마나 크게 반영될 수 있는지 나타내는 감각입니다.

## 자주 헷갈리는 지점

<strong>`inverse(A) @ b`와 `solve(A, b)`를 같은 구현이라고 보는 실수.</strong> 수치적으로는 solve가 더 나은 선택인 경우가 많습니다.

<strong>부동소수점 계산을 실수 계산과 완전히 같다고 생각하는 실수.</strong> 컴퓨터 숫자는 반올림 오차를 가집니다.

<strong>조건수를 정확한 공식으로만 외우는 실수.</strong> 작은 흔들림이 결과에서 커질 수 있는지 보는 감각이 먼저입니다.

<strong>분해를 증명 대상으로만 보는 실수.</strong> 실제로는 안정적이고 재사용 가능한 계산 전략입니다.

## 코드로 읽기

```python
x = np.linalg.solve(A, b)
```

선형시스템을 풀 때는 역행렬을 직접 만들기보다 solve 함수를 쓰는 것이 일반적입니다.

```python
q, r = np.linalg.qr(A)
```

분해는 같은 행렬을 더 계산하기 좋은 구조로 바꾸는 도구입니다.
