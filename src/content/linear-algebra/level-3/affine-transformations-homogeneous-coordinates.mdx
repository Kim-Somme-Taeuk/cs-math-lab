## 왜 배우나

선형변환은 원점을 고정하므로 이동을 자연스럽게 표현하기 어렵습니다. 그래픽스와 게임에서는 이동, 회전, 스케일을 한 transform 흐름으로 묶어야 하므로 아핀 변환과 동차좌표가 필요합니다.

핵심은 <strong>동차좌표를 쓰면 이동까지 행렬 곱 파이프라인에 넣을 수 있다는 것</strong>입니다.

## 먼저 감 잡기

2D 점 `(x, y)`를 `(x, y, 1)`처럼 한 차원 늘려 쓰면, 마지막 열에 이동량을 담아 행렬 곱으로 위치를 옮길 수 있습니다.

## 핵심 정의

### 아핀 변환

선형변환에 이동을 더한 변환입니다. 회전, 스케일, shear, translation을 함께 다룰 수 있습니다.

### 동차좌표

점에 보조 좌표를 하나 붙여 이동까지 행렬 곱으로 표현하게 하는 좌표 표현입니다.

### transform 합성

여러 변환을 행렬 곱으로 묶어 한 번에 적용하는 방식입니다.

### 점과 방향의 w

보통 점은 `w=1`, 방향은 `w=0`으로 두어 이동의 영향을 받을지 구분합니다.

## 자주 헷갈리는 지점

<strong>동차좌표가 실제 3D/4D 공간을 의미한다고 보는 실수.</strong> 계산을 편하게 하는 표현 방식입니다.

<strong>행렬 곱 순서를 무시하는 실수.</strong> 이동 후 회전과 회전 후 이동은 결과가 다릅니다.

<strong>점과 방향을 같은 w로 처리하는 실수.</strong> 방향 벡터는 위치 이동을 받으면 안 되는 경우가 많습니다.

<strong>아핀 변환을 선형변환과 완전히 같다고 보는 실수.</strong> 이동이 포함되면 원점을 고정하지 않습니다.

## 코드로 읽기

```ts
const modelMatrix = translationMatrix.multiply(rotationMatrix).multiply(scaleMatrix);
const worldPoint = modelMatrix.multiply([x, y, 1]);
```

`[x, y, 1]`은 위치 점입니다. 이동 열이 적용되어 실제 위치가 바뀝니다.

```ts
const worldDirection = modelMatrix.multiply([dx, dy, 0]);
```
