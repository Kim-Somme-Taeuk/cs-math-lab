## 왜 배우나

최적화, 물리 시뮬레이션, 머신러닝에서는 여러 입력이 여러 출력에 영향을 줍니다. 그래디언트와 야코비안은 복잡한 함수를 근처에서 선형적으로 읽게 해 줍니다.

핵심은 <strong>비선형 함수도 아주 작은 범위에서는 선형변환처럼 근사해서 변화 방향을 분석할 수 있다는 것</strong>입니다.

## 먼저 감 잡기

언덕 위에서 어느 방향으로 가면 가장 빨리 올라가는지 알려 주는 화살표가 그래디언트입니다. 입력이 여러 개이고 출력도 여러 개라면, 변화 관계를 행렬로 모은 것이 야코비안입니다.

## 핵심 정의

### 그래디언트

여러 변수 함수가 가장 빠르게 증가하는 방향을 나타내는 벡터입니다.

### 야코비안

벡터 입력을 벡터 출력으로 보내는 함수의 국소적인 변화율을 모은 행렬입니다.

### 국소 선형 근사

복잡한 함수를 한 점 근처에서 선형변환처럼 근사하는 관점입니다.

### 최적화 연결

손실 함수를 줄이려면 그래디언트의 반대 방향으로 움직이는 감각이 필요합니다.

## 자주 헷갈리는 지점

<strong>그래디언트를 항상 이동 경로라고 보는 실수.</strong> 그래디언트는 현재 위치에서의 방향 정보입니다.

<strong>야코비안을 그냥 큰 미분표로만 보는 실수.</strong> 입력 변화가 출력 변화로 어떻게 전달되는지 나타내는 선형 근사입니다.

<strong>국소 근사를 전체 함수와 같다고 보는 실수.</strong> 한 점 근처에서만 잘 맞을 수 있습니다.

<strong>자동미분 구현을 먼저 파고드는 실수.</strong> 여기서는 변화 방향과 행렬 감각이 목표입니다.

## 코드로 읽기

```python
gradient = grad(loss_fn)(params)
params = params - learning_rate * gradient
```

그래디언트는 손실을 줄이기 위해 어느 방향으로 파라미터를 바꿀지 알려 줍니다.

```python
local_output_change = jacobian @ input_change
```

야코비안은 작은 입력 변화가 출력 변화로 전달되는 방식을 행렬 곱으로 근사합니다.
