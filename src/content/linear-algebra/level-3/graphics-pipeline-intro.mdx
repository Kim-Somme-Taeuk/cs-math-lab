## 왜 배우나

3D 모델이 화면 픽셀로 보이기까지는 여러 좌표 공간을 거칩니다. 그래픽스 파이프라인을 알면 vertex shader에서 행렬 곱이 반복해서 등장하는 이유를 이해할 수 있습니다.

핵심은 <strong>모델 공간의 점이 월드, 뷰, 클립 공간을 지나 화면에 가까워지는 흐름을 읽는 것</strong>입니다.

## 먼저 감 잡기

모델 파일 안의 점은 오브젝트 기준 좌표입니다. 이 점을 장면에 배치하고, 카메라 기준으로 바꾸고, 화면에 투영해야 우리가 보는 그림이 됩니다.

## 핵심 정의

### 모델 공간

모델 자체를 기준으로 한 좌표입니다. 파일에 저장된 정점 좌표가 보통 여기에 가깝습니다.

### 월드 공간

모델을 장면 안에 배치한 뒤의 좌표입니다.

### 뷰 공간

카메라를 기준으로 본 좌표입니다. 카메라가 원점에 있다고 생각하기 쉽게 바꿉니다.

### 클립 공간

투영 뒤 화면에 보일 후보를 판단하기 위한 공간입니다.

## 자주 헷갈리는 지점

<strong>모든 좌표가 같은 의미라고 보는 실수.</strong> 모델 좌표와 카메라 기준 좌표는 같은 숫자라도 의미가 다릅니다.

<strong>투영을 단순 스케일로만 보는 실수.</strong> 원근 투영은 멀리 있는 물체를 작게 보이게 하는 카메라 모델과 연결됩니다.

<strong>행렬 합성 순서를 대충 보는 실수.</strong> model, view, projection 순서가 파이프라인 의미를 만듭니다.

<strong>vertex shader의 행렬 곱을 암기 코드로만 보는 실수.</strong> 정점을 화면으로 보내는 좌표 변환입니다.

## 코드로 읽기

```glsl
gl_Position = projection * view * model * vec4(position, 1.0);
```

이 한 줄은 모델 좌표의 정점을 월드, 카메라, 투영 좌표로 차례로 보냅니다. 곱셈 순서는 어떤 공간에서 어떤 공간으로 가는지를 나타냅니다.

```ts
const mvp = projectionMatrix.multiply(viewMatrix).multiply(modelMatrix);
```
