# Layout Guidelines

CS Math Lab의 학습 화면은 설명, 조작, 결과를 한 화면 흐름 안에서 빠르게 연결해 보여주는 것을
우선합니다. 새 챕터나 인터랙티브 컴포넌트를 만들 때 아래 기준을 먼저 확인합니다.

## 기본 원칙

- 공백은 보수적으로 크게 잡지 않습니다. 카드 내부는 위아래 여백이 대칭적으로 보이게 맞춥니다.
- 조작부와 결과부는 가능한 가까이 둡니다. 모바일에서 결과가 지나치게 아래로 밀리면 글자를 줄이기보다 보조 정보를 접힘/요약 상태로 옮깁니다.
- `min-height`는 섹션 흔들림을 막기 위해서만 씁니다. 실제 내용보다 큰 빈 공간을 만들면 제거하거나 줄입니다.
- 고정 높이를 쓸 때는 내부 콘텐츠가 잘리지 않는지 `scrollHeight <= clientHeight` 수준으로 확인합니다.
- 데스크톱 2열 레이아웃은 `lg:items-start`를 기본으로 고려합니다. 한쪽 컬럼 높이가 다른 쪽을 강제로 늘리지 않게 합니다.
- 태블릿 768px에서 인터랙티브 섹션이 900px 이상으로 길어지면 `md:grid-cols-*` 전환을 검토합니다. 조작부와 결과부가 한 화면에서 연결되어 보이는 것이 우선입니다.
- 미리보기 영역은 전체 결과를 억지로 다 보여주지 않습니다. 대표 샘플과 `+N개` 요약을 함께 보여주고, 숨김이 의도된 미리보기인지 사용자가 알 수 있게 합니다.

## 본문 콘텐츠

- 코드 블록과 표는 앞뒤 문단에 붙지 않게 충분한 마진을 둡니다.
- 긴 수식은 모바일 폭을 넘을 수 있으므로 KaTeX display 수식은 가로 스크롤을 허용합니다.
- 인라인 코드와 코드 블록은 Cascadia 계열 monospace를 유지합니다.
- 문단 바로 아래에 표, 코드, 퀴즈, 실험이 오는 경우 위쪽 간격이 과하게 뜨지 않는지 확인합니다.

## 인터랙티브 컴포넌트

- 모바일 390px 폭에서 `document.body.scrollWidth`가 viewport보다 커지면 안 됩니다.
- 표처럼 보이는 조작판은 MDX 전역 `table` 스타일 영향을 받을 수 있습니다. 조작 UI에는 실제 `<table>` 대신 CSS grid를 우선 검토합니다.
- 결과 문구, 선택 상태, 칩 개수 변화 때문에 카드 높이가 바뀌지 않게 결과 영역의 높이를 안정화합니다.
- 조건부로 보이는 라벨은 필요하면 항상 같은 자리를 차지하게 하고, 비활성 상태에서는 투명 처리합니다.
- SVG 또는 시각화 패널은 내부 `viewBox` 여백을 확인합니다. 그림 아래 빈 공간이 크면 `viewBox`를 줄입니다.
- 텍스트 화살표(`->`)처럼 임시 느낌이 나는 표현은 배지, 칩, 선, 아이콘 등으로 대체합니다.
- 토글과 세그먼트 버튼은 기능에 비해 크게 만들지 않습니다. 짧은 라벨은 한 줄로 유지하고, 카드형 버튼은 의미 있는 설명이 있을 때만 씁니다.
- 모바일에서 인터랙티브 섹션 자체에 내부 스크롤을 만들지 않습니다. 페이지 스크롤 안에서 자연스럽게 읽히게 하고, 불필요한 빈 공간과 고정 높이를 먼저 줄입니다.
- 모바일에서는 최소 글자 크기를 14px 수준으로 유지합니다. PC 레이아웃을 그대로 축소해 넣는 방식은 글자 가독성을 해치면 쓰지 않습니다.
- 모바일에서 모든 정보를 한 번에 보여주려 하지 않습니다. 핵심 조작과 핵심 결과를 먼저 보이고, 전체 표/상세 판정/계산 흐름은 접힘 영역이나 요약으로 둡니다.
- 태블릿 2열 전환 후에도 텍스트 겹침, SVG 압축, 버튼 줄바꿈이 없는지 768px 캡처로 확인합니다.

## 챕터 이동

- `다음으로` 섹션에는 설명 문장을 길게 두지 않습니다. 기본은 다음 챕터로 가는 버튼 하나입니다.
- 버튼 텍스트와 배경은 실제 캡처에서 읽히는 대비를 기준으로 판단합니다. 어두운 배경에서 글자가 묻히면 밝은 배경과 진한 글자로 대비 방향을 바꿉니다.
- 마지막 ready 챕터처럼 다음 상세 페이지가 없을 때만 로드맵 버튼을 둡니다.

## 퀴즈

- 모바일에서 여러 문제를 한 섹션에 모두 펼치면 지나치게 길어질 수 있습니다. 문제 목록 영역은 내부 스크롤 제한을 둡니다.
- 선택지 패딩은 모바일에서 과하게 크지 않게 잡고, 데스크톱에서만 여유를 늘립니다.
- 채점 후 해설이 생겨도 카드 높이 변화가 과하지 않은지 확인합니다.

## 검증 체크리스트

- `npm run lint`
- `npm run test`
- `npm run build`
- 큰 레이아웃 변경이면 `npm run test:e2e`
- 모바일 폭 390px에서 주요 챕터의 `body.scrollWidth`가 viewport와 같은지 확인
- 긴 상태, 빈 상태, 최악 상태를 직접 클릭해서 결과 박스 잘림과 섹션 흔들림 확인
