# CS Math Lab

CS Math Lab은 컴퓨터공학 전공자가 전공 학습에 필요한 수학 기초를 단계적으로 익힐 수
있도록 돕는 인터랙티브 웹사이트입니다. 장기적으로는 학습자 상태를 추적하고, 목표와 약점에
맞춰 로드맵, 문제, 설명을 조정하는 초개인화 AI 학습 플랫폼을 지향합니다.

전체 구조는 이산수학, 선형대수, 미적분, 확률통계 대주제로 나누고, 현재 MVP는 이산수학
Level 1 입문과 Level 2 일부 확장 범위에 집중합니다.

## 기능

- 홈 페이지와 컴공 수학 로드맵
- 과목별 학습 트랙 페이지(`/subjects/[subjectSlug]`)
- MDX 기반 챕터 콘텐츠
- KaTeX 수식 렌더링
- 논리 조건식 빌더
- 조건문 위반 탐지기
- 집합 필터 실험실 및 벤 다이어그램
- 함수 매핑 직접 조작기
- 관계 성질 판별기
- 귀납법 연결 시뮬레이터
- 경우의 수 시뮬레이터
- DFS/BFS 단계 실행기
- 챕터별 퀴즈와 즉시 피드백
- 템플릿 기반 변형 종합 점검
- 학습 목표, 수준, 선호 기반 개인화 추천 경로
- 퀴즈 결과와 이해도 체크 기반 약점/복습 추천
- 챕터 끝 다음 학습 이동 링크
- 과목 상세 페이지의 챕터 카드 그래픽
- 데스크톱 챕터 사이드바와 모바일 햄버거형 챕터 이동 패널
- 반응형 UI

## 로드맵 범위

최상위 대주제:

- 이산수학
- 선형대수
- 미적분
- 확률통계

로드맵은 전체 챕터를 한 화면에 길게 펼치는 대신 대주제 카드 중심으로 보여줍니다.
대주제를 선택하면 `/subjects/discrete-math` 같은 과목 상세 페이지로 이동하고, 그 안에서
Level별 챕터 묶음과 추천 시작점을 확인합니다.

각 대주제 안에는 Level이 있고, 각 Level 안에 챕터가 있습니다. 현재 상세 페이지가 열리는
활성 범위는 이산수학 Level 1 ready 챕터입니다.

이산수학의 현재 8개 챕터는 `Level 1. 입문`입니다. 전체 이산수학 커리큘럼이 아니라
전공 학습에 필요한 최소 입문 범위를 먼저 다루는 MVP입니다.

이산수학 Level 1 ready 콘텐츠:

- 명제와 논리
- 조건문과 진리표
- 집합과 집합 연산
- 함수
- 관계
- 수학적 귀납법
- 경우의 수
- 그래프 기초

이산수학 향후 확장 예정 레벨:

- `Level 2. 핵심 확장`: 증명 기법, 술어 논리, 동치관계, 점화식, 포함배제 원리 등
- `Level 3. 컴공 응용`: 알고리즘 분석, 이산확률, 정수론, 부울 대수, DAG와 위상 정렬 등

선형대수, 미적분, 확률통계는 각각 Level 1/2/3 구조와 planned 챕터 목록을 과목 상세 페이지에
표시하고, 세부 챕터 페이지는 아직 만들지 않습니다.

예정 챕터는 과목 상세 페이지에서 예정으로 표시하고 클릭되지 않습니다. 일반 챕터 이동과 상세
페이지 접근에서는 ready 챕터만 노출합니다.

## 콘텐츠 원칙

- 기본 코드 예시는 Python을 먼저 사용합니다.
- TypeScript/JavaScript 예시는 필요한 경우에만 보조로 둡니다.
- 코드 전에는 자연어 상황과 수학적 분해를 먼저 제시합니다.
- 코드 후에는 수학 개념과 코드 표현의 대응 관계를 짧게 설명합니다.
- 코드 예시는 문법 학습이 아니라 수학 개념 연결을 위한 보조 자료로 사용합니다.

## 개인화 학습 방향

현재 개인화 기능은 로그인이나 DB 없이 브라우저 `localStorage`를 사용합니다. 사용자는
로드맵에서 목표, 현재 수준, 선호 학습 방식을 고르고, 사이트는 공개된 ready 챕터 안에서
추천 경로를 조정합니다.

저장하는 학습 상태:

- 학습 목표, 수준, 선호
- 완료한 챕터
- 퀴즈 점수와 관련 개념 태그
- 틀린 문제의 개념 태그와 문제 유형
- "이해했음 / 아직 헷갈림" 이해도 체크

이 상태를 바탕으로 로드맵은 약점 개념, 약한 문제 유형, 복습 추천, 다음 학습 코치 메모를
보여줍니다. 챕터 데이터에는 선행 챕터, 개념 태그, 학습 트랙 태그를 붙여 두어 이후 선형대수,
미적분, 확률통계가 ready로 열려도 같은 추천 구조를 확장할 수 있게 합니다.

`src/lib/aiLearningContext.ts`는 AI 코치가 사용할 최소 학습 컨텍스트를 만듭니다. AI는
검증되지 않은 새 개념을 마음대로 생성하는 역할이 아니라, 현재 챕터와 저장된 학습 상태를
바탕으로 설명을 바꾸거나 추가 연습 문제를 제안하는 보조 레이어로 붙일 계획입니다.

AI 코치 MVP는 `/roadmap` 개인화 패널의 "AI 코치 메모 생성" 버튼에서 시작합니다. 서버
라우트 `POST /api/ai/coach`가 `OPENAI_API_KEY`를 읽어 OpenAI Responses API를 호출하며,
클라이언트에는 API 키가 전달되지 않습니다. 같은 학습 컨텍스트에 대해서는 브라우저
`localStorage`에 저장된 최근 코치 메모를 재사용해 불필요한 반복 호출을 줄입니다.

필요한 서버 환경변수:

- `OPENAI_API_KEY`: OpenAI API 호출용 서버 전용 키
- `OPENAI_MODEL`: 선택 사항, 기본값은 `gpt-5.2`

AI 코치 요청에 전송되는 데이터:

- 학습 목표, 수준, 선호를 요약한 문자열
- 완료한 챕터 slug
- 추천 다음 챕터 slug
- 복습 추천 챕터 slug
- 약점 개념과 약한 문제 유형
- 최근 퀴즈 점수 비율과 오답 태그
- ready 챕터의 slug, 제목, 과목, level, 선행 챕터, 개념 태그, 트랙 태그

로그인 정보, 이름, 이메일, 브라우저 식별자, 원문 대화 로그는 전송하지 않습니다. 서버는
payload 길이와 필드를 검증하고, API 키가 없거나 호출에 실패하면 고정 fallback 코치 메모를
반환합니다.

집합 챕터의 종합 점검은 고정 문제 대신 템플릿 기반 변형 문제를 사용합니다. 사용자가 문제를
틀리면 같은 개념을 유지하되 숫자와 집합 구성이 바뀐 비슷한 문제를 다시 풀 수 있습니다.

## 레이아웃 원칙

학습 본문, 인터랙티브 컴포넌트, 퀴즈의 여백과 모바일 동작 기준은
[docs/layout-guidelines.md](docs/layout-guidelines.md)를 따릅니다. 인터랙티브 섹션은 모바일
가로 밀림을 만들지 않고, 실험 박스 내부 스크롤 대신 페이지 흐름 안에서 읽히도록 설계합니다.
PC 레이아웃은 유지하되 모바일에서는 중복 정보와 보조 설명을 접힘/요약 상태로 줄여 핵심 조작과
결과를 먼저 보여줍니다.

최근 모바일 개선 사항:

- 모바일 챕터 본문이 화면 폭을 더 넓게 쓰도록 외곽 카드 여백을 축소했습니다.
- 모바일에서는 하단 중복 사이드바 대신 오른쪽 아래 햄버거형 챕터 이동 패널을 사용합니다.
- 논리 조건식 빌더는 모바일에서 결과 패널과 값/조건식 조작 패널을 나란히 보여줍니다.
- 조건문, 집합 실험은 모바일에서 보조 표/상세 판정을 접힘 영역으로 이동했습니다.
- 그래프 탐색 실험은 모바일에서 탐색 컨트롤과 상태 패널 공백을 줄였습니다.

## 실행 방법

```bash
npm install
npm run dev
```

`dev` 스크립트는 개발 서버 출력 폴더를 `.next-dev`로 분리해 실행합니다. 일반 Next.js 기본
동작 그대로 확인해야 할 때는 `dev:default`를 사용합니다.

```bash
npm run dev:default
```

이 작업환경에서는 `plana-ai`가 `localhost:3000`을 사용하므로, CS Math Lab을 확인할 때는
다음처럼 3001 포트로 실행합니다.

```bash
npm run dev -- --hostname 0.0.0.0 --port 3001
```

브라우저에서 `http://localhost:3001`로 접속합니다.

주의: `localhost:3001` 접근은 앱 코드가 아니라 ccc/Docker 포트 포워딩 설정에 의존합니다.
앱 서버가 컨테이너 내부에서 응답해도 호스트 포워딩이 없으면 브라우저의 `localhost:3001`은
열리지 않을 수 있습니다.
